# `const entries = await readdir(docPath, { withFileTypes: true });`

## 一句话解释
**读取指定目录（docPath）下的所有文件/文件夹，并返回带类型信息的列表，方便直接判断是文件还是文件夹。**

---

## 逐段拆解（超清晰）
### 1. `await readdir(...)`
- `readdir` = **read directory**（读取目录）
- 是 Node.js 文件系统的异步方法
- 前面加 `await` → 异步等待读取完成

### 2. `docPath`
- 你要读取的**目录路径**
- 就是上一行代码得到的：`项目根目录/docs`

### 3. `{ withFileTypes: true }`（核心！）
- **返回的每个条目自带类型**
- 不用再调用 `stat` 就能知道：
  - 是**文件**？
  - 还是**文件夹**？
- 性能更高、代码更简洁

### 4. `entries`
- 读取到的**目录内容列表**（数组）
- 每个元素包含：`name`（名称） + `type`（类型）

---

## 拿到 entries 后怎么用？（最常用代码）
```javascript
const entries = await readdir(docPath, { withFileTypes: true });

for (const entry of entries) {
  const name = entry.name; // 文件名/文件夹名

  if (entry.isDirectory()) {
    console.log('📁 文件夹：', name);
  } else if (entry.isFile()) {
    console.log('📄 文件：', name);
  }
}
```

### 不用再 stat！直接判断类型 ✅

## 真实输出示例
假设 docs 目录里有：
- guide（文件夹）
- README.md（文件）
- config.js（文件）

那么 `entries` 会返回：
```
[
  { name: 'guide', isDirectory: () => true, ... },
  { name: 'README.md', isFile: () => true, ... },
  { name: 'config.js', isFile: () => true, ... }
]
```

- `await` → 异步等待
- `entries` → 目录列表（可直接判断文件/文件夹）
