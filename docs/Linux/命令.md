# 译文：Linux 下管理 Nginx 必备核心命令
在 Linux 上使用 Nginx 时，掌握一些核心命令能大幅简化服务器管理。以下是最实用的基础命令及其用途：

## 一、目录与文件管理
- **cd**：切换目录
  示例：`cd /etc/nginx` 进入 Nginx 配置文件夹
- **ls**：列出目录内容
  示例：`ls -la` 显示文件详细信息（包括隐藏文件）
- **cp / mv**：复制、移动文件或目录
  示例：`cp nginx.conf nginx.conf.bak` 备份配置文件
- **rm**：删除文件或目录
  示例：`rm nginx.conf` 删除配置文件
  示例：`rm -r conf.d` 递归删除目录及内部所有文件

## 二、查看与编辑文件
- **cat**：显示文件全部内容
  示例：`cat /etc/nginx/nginx.conf`
- **less**：分页查看大文件（日志/配置）
  示例：`less /var/log/nginx/error.log`
- **tail**：查看文件末尾，实时监控日志
  示例：`tail -f /var/log/nginx/access.log` 实时看访问日志
- **vim / nano**：终端文本编辑器
  示例：`vim /etc/nginx/nginx.conf` 编辑配置

## 三、进程与服务管理
- **sudo**：以管理员权限执行命令
  示例：`sudo systemctl restart nginx`
- **systemctl**：管理系统服务（现代 Linux 标准）
  示例：
  `sudo systemctl start nginx`   启动
  `sudo systemctl stop nginx`    停止
  `sudo systemctl restart nginx` 重启
  `sudo systemctl status nginx`  查看运行状态
- **service**：部分老版 Linux 系统使用
  示例：`sudo service nginx status`

## 四、网络与端口工具
- **netstat / ss**：查看端口与网络连接
  示例：`sudo netstat -tulpn | grep nginx` 查看 Nginx 监听端口
- **curl**：发送 HTTP 请求，测试服务
  示例：`curl -I http://localhost` 查看 Nginx 返回的响应头

## 五、搜索与进程监控
- **grep**：在文件中搜索关键词
  示例：`grep 'error' /var/log/nginx/error.log` 搜索错误日志
- **ps / kill**：查看进程、强制结束进程
  示例：`ps aux | grep nginx` 查看 Nginx 进程
  示例：`sudo kill <PID>` 结束异常进程

---

这些命令是在 Linux 上高效管理 Nginx 的基础，能帮你快速操作文件、管理服务、排查问题。

# 📄 Nginx 必备 Linux 命令 · 一页速查清单（可直接保存）
## 1️⃣ 目录与文件（配置/备份/删除）
```bash
cd /etc/nginx                # 进入Nginx配置目录
ls -la                      # 查看所有文件（含隐藏）
cp nginx.conf nginx.conf.bak # 备份配置
mv old.conf new.conf         # 重命名/移动文件
rm nginx.conf                # 删除文件
rm -r conf.d                 # 删除整个目录
```

## 2️⃣ 查看文件 / 日志
```bash
cat nginx.conf               # 直接查看文件全文
less error.log                # 分页查看大文件
tail -f access.log            # 实时刷新查看访问日志
```

## 3️⃣ 编辑配置
```bash
vim /etc/nginx/nginx.conf     # 编辑配置（vim）
nano /etc/nginx/nginx.conf    # 编辑配置（nano 简单版）
```

## 4️⃣ Nginx 服务管理（最常用）
```bash
nginx -t                     # 检查配置语法是否正确
nginx -s reload               # 平滑重载配置（推荐）
nginx -s restart              # 硬重启（会断连接）

systemctl start nginx         # 启动
systemctl stop nginx          # 停止
systemctl restart nginx       # 重启
systemctl status nginx        # 查看运行状态
systemctl enable nginx        # 开机自启
```

## 5️⃣ 查看进程 / 端口
```bash
ps aux | grep nginx           # 查看Nginx进程
kill -9 <PID>                # 强制杀死进程
netstat -tulpn | grep nginx   # 查看监听端口
ss -tulpn | grep nginx        # 新版查看端口（替代netstat）
```

## 6️⃣ 测试请求 / 验证服务
```bash
curl -I http://localhost      # 查看响应头，测试Nginx是否正常
curl http://127.0.0.1         # 直接访问本机
```

## 7️⃣ 搜索日志/文件内容
```bash
grep 'error' error.log        # 在日志里搜错误
grep '404' access.log        # 搜404请求
```
