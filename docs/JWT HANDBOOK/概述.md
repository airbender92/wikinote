这份《JWT Handbook》是由Auth0的Sebastián E. Peyrott编写的权威JWT技术手册，**完整覆盖JWT核心原理、结构、算法、应用、攻击与安全最佳实践**，以下是精炼总结：

# 一、核心定位
JWT（JSON Web Token）是**轻量、紧凑、URL安全**的标准，用于在多方间安全传输声明（Claims），支持**签名（JWS）**与**加密（JWE）**，是认证、授权、联合身份、无状态会话的主流方案。

# 二、基础结构
JWT采用**Base64URL编码**，以`.`分隔三部分：
1. **Header（头部）**：声明算法（alg）、类型（typ）等
2. **Payload（载荷）**：存放标准声明/自定义声明
3. **Signature/Encrypted Data（签名/加密数据）**：保障完整性与机密性

---

# 三、标准声明（核心7个）
- `iss`：签发者
- `sub`：主题/用户唯一标识
- `aud`：受众/接收方
- `exp`：过期时间
- `nbf`：生效时间
- `iat`：签发时间
- `jti`：唯一ID，防重放

---

# 四、两大核心形态
## 1. JWS（JSON Web Signature，签名）
- 作用：**防篡改、验真伪**，不加密内容
- 必选算法：**HS256**（HMAC-SHA256，共享密钥）
- 推荐算法：**RS256**（RSA）、**ES256**（椭圆曲线）
- 结构：Header.Payload.Signature

## 2. JWE（JSON Web Encryption，加密）
- 作用：**内容保密**，仅授权方可解密
- 结构：5段式（头部、加密密钥、初始向量、密文、认证标签）
- 常用组合：RSA-OAEP/ECDH 加密密钥 + AES-GCM 加密内容

---

# 五、关键配套标准
- **JWK**：JSON Web Key，统一密钥格式，支持RSA/EC/oct等密钥
- **JWKS**：JWK集合，方便公钥分发
- **JWA**：JSON Web Algorithms，定义所有加密/签名算法规范

---

# 六、主流应用场景
1. **用户认证与授权**：登录后签发JWT，接口鉴权
2. **无状态会话**：客户端存储会话数据，服务端不存Session
3. **联合身份/SSO**：OpenID Connect 核心载体，跨系统信任登录
4. **API安全**：OAuth2.0 访问令牌/刷新令牌

---

# 七、典型攻击与风险
1. **alg:none 攻击**：移除签名、设算法为none，绕过校验
2. **RS256公钥当HS256密钥攻击**：算法混淆导致伪造签名
3. **弱HMAC密钥**：密钥过短易被暴力破解
4. **无效椭圆曲线攻击**：非法曲线点泄露私钥
5. **令牌替换攻击**：跨服务/跨接口盗用合法令牌
6. **XSS/CSRF**：窃取存储在Cookie/LocalStorage的JWT

---

# 八、安全最佳实践（核心）
1. **强制校验算法**，禁止依赖header中的alg
2. **优先非对称算法**（RS256/ES256），避免共享密钥风险
3. **使用强密钥**：HS256密钥≥256位，随机生成
4. **校验所有声明**：`iss/aud/exp/nbf` 必验
5. **Cookie设置HttpOnly/Secure**，防XSS窃取
6. **短有效期**，配合刷新令牌使用
7. **校验公钥合法性**（椭圆曲线必验点有效性）
8. **不同场景用不同密钥**，避免令牌混用

---

# 九、核心结论
JWT简单易用但**安全门槛高**，核心风险来自**实现漏洞**而非协议本身；严格遵循校验规则、选用成熟库、做好密钥管理，才能安全落地。
