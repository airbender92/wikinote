# JWK 到底是什么？（一句话核心）
**JWK 就是用 JSON 格式来表示密钥的标准。**
不管是 RSA、EC（椭圆曲线）、还是 AES 密钥，
全都能写成一段干净的 JSON。

---

# 为什么要有 JWK？
以前密钥是这种样子：
```
-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcw...
-----END PRIVATE KEY-----
```
又长又怪，不适合在网络里传。

**JWK 把密钥变成 JSON**：
- 好读
- 好传
- 好解析
- 能直接放进 JWT / JWE / JWS 里用

---

# JWK 长什么样？（一眼看懂）
```json
{
  "kty": "EC",
  "crv": "P-256",
  "x": "...",
  "y": "...",
  "d": "...",
  "use": "enc",
  "kid": "1"
}
```

## 只需要记住 4 个关键字段
1. **kty**：密钥类型（RSA / EC / oct）
2. **use**：用途（sig=签名 / enc=加密）
3. **kid**：密钥ID（用来快速找密钥）
4. **x/y/d**：公钥/私钥内容

---

# JWK 有什么用？（实际用途）
1. **公钥可以公开暴露**（比如 https://xxx/.well-known/jwks.json）
2. 别人拿到你的 JWK，**直接就能验证你的 JWT**
3. 不用再传 PEM 文件、不用配置证书
4. 微服务、分布式系统**自动换密钥、自动拉公钥**

---

# JWK Set（JWKS）是什么？
就是**一堆 JWK 放在一起**，方便同时管理多套密钥。

```json
{
  "keys": [
    { "kty": "EC", ... },
    { "kty": "RSA", ... }
  ]
}
```

---

# 最精简总结（你只需要记这个）
- **JWK = JSON 格式的密钥**
- **JWKS = 一组 JWK**
- 作用：**统一密钥格式、方便分发、自动验签**
- 是现代 JWT 认证体系的**标配**

# 6.1 JWK 结构（超级精简版）
**JWK 就是一个 JSON 对象，用来描述一个密钥的所有信息。**
不同类型的密钥（EC / RSA / 对称）有不同的参数，但**公共字段是固定的**。

---

## 一、必填字段（只有 1 个）
### **kty (key type) 密钥类型**
表示这是什么密钥：
- **EC**：椭圆曲线密钥
- **RSA**：RSA密钥
- **oct**：对称密钥（如AES）

---

## 二、常用可选字段（最重要）
### **use**：用途
- **sig**：用于签名（JWS）
- **enc**：用于加密（JWE）
不写 = 可用于所有场景

### **key_ops**：具体操作权限（更细粒度）
如：sign、verify、encrypt、decrypt、wrapKey 等
**和 use 二选一，不要同时用**。

### **alg**：推荐使用的算法
如 HS256、RS256、ES256、A128GCM 等

### **kid**：密钥ID
用来快速识别、查找密钥
尤其在密钥轮换（多个密钥并存）时非常重要

---

## 三、X.509 证书相关（用于兼容传统证书）
### **x5u**
URL 地址，指向 X.509 证书

### **x5c**
证书内容本身（Base64URL 格式）

### **x5t / x5t#S256**
证书的指纹（哈希），用于校验证书真伪

---

## 四、每种密钥自己的专属参数
- **EC 密钥**：有 `crv`, `x`, `y`, `d`
- **RSA 密钥**：有 `n`, `e`, `dp`, `dq` 等
- **oct 对称密钥**：有 `k`

这些是密钥的数学参数，不用背，库会自动处理。

---

# 最终超精简总结（你只需要记这个）
**JWK = 用 JSON 表示密钥**
- **必填：kty（类型：EC/RSA/oct）**
- **常用：use（用途）、alg（算法）、kid（密钥ID）**
- **兼容X.509：x5u、x5c、x5t**
- **专属参数**：不同密钥各不相同

---

# 6.1.1 JSON Web Key Set（一句话概括）
**JWK Set 就是一组 JWK 密钥放在同一个 JSON 里，方便统一管理、对外暴露公钥。**

---

# 核心内容（精简讲透）
- JWK 规范允许把**多个密钥归为一组**，这一组就叫 **JWK Set**。
- 组内密钥的含义、顺序，由使用者自己定义，规范没有强制要求。
- 结构非常简单：一个 JSON 对象，里面只有一个 **`keys` 数组**，数组里每一项都是一个 JWK。

---

# 示例里有什么？
里面放了**两个公钥**：
1. **EC 椭圆曲线密钥**
   - 类型：EC
   - 用途：`use: "enc"` → 只用于**加密（JWE）**
2. **RSA 密钥**
   - 类型：RSA
   - 指定算法：`alg: "RS256"` → 专门用于**签名（JWS）**

---

# 实际用处（最关键）
- 服务端可以把所有公钥放在一个接口返回，比如 `/oauth2/jwks`
- 客户端拿到这个 JWK Set，会根据 JWT 头部的 **kid** 自动找到对应公钥验签
- 支持**密钥轮换**：旧密钥不删、新密钥加进去，不影响正在使用的 token

---

# 超精简总结
- **JWK = 单个密钥**
- **JWK Set = 多个 JWK 放在一个 JSON 里**
- 结构固定：`{ "keys": [ JWK, JWK, ... ] }`
- 主要用于**集中管理公钥、自动验签、密钥轮换**
