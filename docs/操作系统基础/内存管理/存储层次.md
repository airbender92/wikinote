# 存储器层次结构（标准背诵版）
## 一、层次结构（从上到下）
**寄存器 → 高速缓存（Cache）→ 主存（内存）→ 辅存（外存）**

---

## 二、每层特点（速度、容量、价格、CPU访问方式）
### 1. 寄存器（Register）
- 位置：**CPU 内部**
- 速度：**最快**
- 容量：最小（几 KB 以内）
- 成本：最贵
- 作用：存放当前立即要使用的数据、指令
- 与 CPU：**直接访问**

### 2. 高速缓存 Cache
- 位置：CPU 与主存之间
- 速度：略慢于寄存器，远快于内存
- 容量：较小（几 MB～几十 MB）
- 作用：**缓解 CPU 和内存速度不匹配**，存放近期常用数据
- 原理：**局部性原理**（时间局部性、空间局部性）

### 3. 主存（内存 / RAM）
- 位置：主板上
- 速度：中等
- 容量：GB 级
- 作用：运行时程序和数据存放的地方
- 特点：断电丢失数据（易失性）
- CPU 可以**直接访问**

### 4. 辅存（外存）
- 如：硬盘、U盘、光盘
- 速度：最慢
- 容量：最大（TB 级）
- 成本：最便宜
- 特点：断电不丢失（非易失性）
- CPU **不能直接访问**，必须先调入内存才能执行

---

## 三、整体规律（一句话记住）
从上到下：
**速度越来越慢，容量越来越大，价格越来越低，CPU访问越来越远。**

---

## 四、常考要点
1. **CPU 能直接访问的：寄存器、Cache、主存**
2. **辅存必须调入主存才能被 CPU 访问**
3. 引入层次结构的目的：
   - 提高存储系统**性价比**
   - 接近上层的速度，拥有下层的容量
4. 局部性原理是 Cache 能生效的关键
