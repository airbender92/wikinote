# 逻辑地址 → 物理地址（操作系统必考精简版）

## 一、基本概念
- **逻辑地址（相对地址 / 虚拟地址）**
  - 程序编译、链接后生成的地址
  - 从 0 开始编址，是**程序员看到的地址**
- **物理地址（绝对地址）**
  - 内存单元的真实地址
  - CPU 寻址时使用的地址
- **地址转换（重定位）**
  - 把逻辑地址翻译成物理地址的过程
  - 由 **MMU（内存管理单元）** 硬件完成

---

# 二、三种经典转换方式

## 1. 绝对装入（编译时装入）
- 编译时就确定物理地址
- 程序只能放在**固定内存位置**
- 无地址转换过程

## 2. 静态重定位（装入时转换）
- 装入内存时，一次性把所有逻辑地址改成物理地址
- 公式：
  **物理地址 = 逻辑地址 + 起始地址**
- 优点：简单，无需硬件
- 缺点：装入后**不能移动**，内存利用率低

## 3. 动态重定位（运行时转换）
- 装入时不修改，**执行指令时才转换**
- 硬件支持：**重定位寄存器（基址寄存器 BR）**
- 公式：
  **物理地址 = 逻辑地址 + 基址寄存器值**
- 优点：程序可在内存移动、支持虚拟内存
- 现代操作系统通用方式

---

# 三、连续分配方式的地址转换
## 单一连续分配
物理地址 = 逻辑地址 + 内存起始地址

## 固定分区 / 动态分区
物理地址 = 逻辑地址 + 分区起始地址

---

# 四、分页存储的地址转换（重点！）
## 1. 地址结构
逻辑地址分为两部分：
**页号 P + 页内偏移量 W**

## 2. 转换步骤
1. 用逻辑地址算出：
   - 页号 P = 逻辑地址 / 页面大小
   - 页内偏移 W = 逻辑地址 % 页面大小
2. 查**页表**，得到该页对应的**物理块号 f**
3. 物理地址 = f × 页面大小 + W

> 口诀：页号查页表得块号，块号拼偏移得物理地址

---

# 五、分段存储的地址转换
## 1. 地址结构
**段号 S + 段内偏移 W**

## 2. 转换步骤
1. 查**段表**，得到该段的**内存起始地址（基址）**
2. 判断偏移是否越界
3. 物理地址 = 段基址 + 段内偏移

---

# 六、段页式（了解）
- 先分段，再分页
- 逻辑地址：段号 → 页号 → 页内偏移
- 先查段表得页表始址，再查页表得物理块号，最后拼偏移

---

# 七、一句话总结
- 连续分配：**+ 基地址**
- 分页：**页号查页表 → 块号 + 偏移**
- 分段：**段号查段表 → 基址 + 偏移**
- 全部由 **MMU 硬件** 完成，对用户透明
