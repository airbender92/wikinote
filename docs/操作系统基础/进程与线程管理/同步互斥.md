# 操作系统·同步互斥核心总结（考试/面试直接背）

## 一、基本概念
- **同步**：进程间按**先后顺序**协作（前操作完成，后操作才能执行）
- **互斥**：多个进程**不能同时进入临界区**（竞争同一临界资源）
- **临界区**：访问临界资源的代码段
- **临界资源**：一次只允许一个进程使用的资源（打印机、变量、缓冲区等）

# 二、信号量机制（重点）
## 1. 信号量定义
- 一个**整型变量 + 等待队列**
- 只能通过 **P、V 原语** 操作（原子、不可中断）

### P 操作（申请资源）
```
P(S):
S = S - 1
if S < 0: 阻塞，进入等待队列
```

### V 操作（释放资源）
```
V(S):
S = S + 1
if S ≤ 0: 唤醒等待队列中一个进程
```

---

## 2. 信号量分类
1. **互斥信号量**
   - 初值 = 1
   - 用于临界区互斥访问
   - 结构：
     ```
     P(mutex);
     临界区;
     V(mutex);
     ```

2. **同步信号量**
   - 初值 = 0 或某个正整数
   - 用于控制执行顺序

---

# 三、管程（Monitor）
- 一种**高级同步机制**，把**共享数据 + 操作过程**封装在一起
- 特点：
  1. 一次**只允许一个进程**进入管程
  2. 内部自动实现互斥
  3. 用 **condition 变量 + wait / signal** 实现同步
- 优点：比信号量更安全、不易死锁、编写简单

# 四、经典同步互斥问题（必考）

## 1）生产者—消费者问题（缓冲池有界）
### 条件
- n 个缓冲区
- 生产者放产品
- 消费者取产品
- 互斥访问缓冲池
- 不能往满缓冲区放，不能从空缓冲区取

### 信号量设置
- `mutex = 1`      互斥访问缓冲池
- `empty = n`      空缓冲区数量
- `full = 0`       满缓冲区数量

### 生产者
```
while(1){
    生产产品;
    P(empty);
    P(mutex);
    放入缓冲区;
    V(mutex);
    V(full);
}
```

### 消费者
```
while(1){
    P(full);
    P(mutex);
    从缓冲区取产品;
    V(mutex);
    V(empty);
    使用产品;
}
```

> 要点：**同步 P 操作在前，互斥 P 操作在后**，否则容易死锁。

## 2）哲学家进餐问题
5 个哲学家围坐，左右各一根筷子，共 5 根；
想吃饭必须拿起**左右两根筷子**。

### 问题隐患
- 所有人同时拿起左边筷子 → **死锁**

### 解法1：最多允许4人同时抢筷子（最常用）
- 信号量：
  - `mutex[5] = {1,1,1,1,1}`  5根筷子
  - `count = 4`              限制就餐人数

### 哲学家 i
```
while(1){
    P(count);
    P(mutex[i]);       // 拿左筷子
    P(mutex[(i+1)%5]); // 拿右筷子
    吃饭;
    V(mutex[i]);
    V(mutex[(i+1)%5]);
    V(count);
    思考;
}
```

### 解法2：奇数号先左后右，偶数号先右后左
破坏循环等待，避免死锁。
