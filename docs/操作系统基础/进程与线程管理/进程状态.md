下面用**最清晰、考试/面试标准版本**给你总结操作系统中进程的**三种基本状态 + 状态转换**。

# 一、进程三种基本状态
1. **就绪状态（Ready）**
   - 已获得除CPU外的所有资源
   - 只要分配到CPU就能立即执行

2. **运行状态（Running）**
   - 进程正在CPU上执行
   - 单核CPU同一时刻**只有一个**进程运行

3. **阻塞状态（Blocked / Waiting）**
   - 因等待某事件（I/O、信号、资源）暂时无法运行
   - 即使CPU空闲也不能运行

---

# 二、状态转换图（必考）
## 1. 就绪 → 运行
- 发生：**调度程序选中该进程**（调度/分派）

## 2. 运行 → 就绪
- 发生：
  - **时间片用完**（抢占式调度）
  - 被更高优先级进程抢占

## 3. 运行 → 阻塞
- 发生：进程**主动等待某事件**
  - 等待I/O（读磁盘、键盘）
  - 等待信号量、锁、消息

## 4. 阻塞 → 就绪
- 发生：**等待的事件完成**
  - I/O结束
  - 获得资源
  - 收到信号/中断唤醒

---

# 三、一句话记忆流程
> **就绪 →（调度）→ 运行 →（时间片/抢占）→ 就绪**
> **运行 →（等待事件）→ 阻塞 →（事件完成）→ 就绪**

⚠️ 注意：
- **阻塞不能直接到运行**
- **就绪不能直接到阻塞**
- 只有运行态能进入阻塞

---

# 四、扩展：常见补充状态（了解）
- **创建状态（New）**：进程刚被创建，还未提交就绪队列
- **终止状态（Terminated）**：运行结束，等待系统回收资源
