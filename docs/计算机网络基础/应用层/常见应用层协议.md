## 一、什么是应用层？

**应用层（Application Layer）是 OSI 模型和 TCP/IP 模型中的最高一层**，直接为用户和应用程序提供服务。

✅ 主要功能：
- 为用户应用程序提供网络服务
- 定义**应用之间如何通信**
- 规定**数据格式、交互规则（协议）**

📌 应用层协议通常运行在：
- **TCP**（可靠传输）：HTTP、HTTPS、FTP、SMTP、POP3、IMAP
- **UDP**（不可靠、速度快）：DNS、DHCP

---

## 二、常见应用层协议详解

### 1️⃣ HTTP / HTTPS（网页访问）

#### HTTP（HyperText Transfer Protocol）
- 作用：**传输网页数据**
- 默认端口：**80**
- 特点：
  - 基于 TCP
  - 明文传输（不安全）

#### HTTPS（HTTP + SSL/TLS）
- 作用：加密的网页访问
- 默认端口：**443**
- 特点：
  - 安全性高
  - 用于登录、支付等场景

✅ 常见用途：
- 浏览器访问网站
- 前后端数据交互（接口）

---

### 2️⃣ DNS（域名系统）

- 作用：**域名 → IP 地址**
- 协议：**UDP（常用） / TCP**
- 端口：**53**

✅ 举例：
```
www.baidu.com → 110.242.68.66
```

✅ 特点：
- 分布式数据库
- 上网第一步就是 DNS 解析

---

### 3️⃣ FTP（文件传输协议）

- 作用：**上传/下载文件**
- 协议：**TCP**
- 端口：
  - 控制连接：**21**
  - 数据连接：**20**

✅ 常见用途：
- 网站服务器上传文件
- 企业内部文件共享

⚠️ 明文传输，安全性较差

---

### 4️⃣ SMTP / POP3 / IMAP（电子邮件协议）

#### ✉️ SMTP（Simple Mail Transfer Protocol）
- 作用：**发送邮件**
- 协议：**TCP**
- 端口：**25**

#### 📥 POP3（Post Office Protocol v3）
- 作用：**下载邮件到本地**
- 端口：**110**
- 特点：
  - 下载后服务器通常删除邮件

#### 📂 IMAP（Internet Message Access Protocol）
- 作用：**在线管理邮件**
- 端口：**143**
- 特点：
  - 邮件保留在服务器
  - 多设备同步

✅ 对比：
| 协议 | 功能 | 是否保留服务器 |
|----|----|----|
| SMTP | 发邮件 | — |
| POP3 | 收邮件 | ❌ |
| IMAP | 收邮件 | ✅ |

---

### 5️⃣ DHCP（动态主机配置协议）

- 作用：**自动分配 IP 地址**
- 协议：**UDP**
- 端口：
  - 客户端：68
  - 服务器：67

✅ 自动分配内容包括：
- IP 地址
- 子网掩码
- 网关
- DNS 服务器

✅ 特点：
- 即插即用
- 避免 IP 冲突

---

### 6️⃣ Telnet / SSH（远程登录）

#### Telnet
- 作用：远程登录主机
- 端口：**23**
- 特点：
  - 明文传输
  - 不安全

#### SSH（Secure Shell）
- 作用：安全远程登录
- 端口：**22**
- 特点：
  - 加密传输
  - 常用于服务器管理

✅ 对比：
| 协议 | 安全性 | 使用场景 |
|----|----|----|
| Telnet | ❌ | 已基本淘汰 |
| SSH | ✅ | Linux 远程管理 |
