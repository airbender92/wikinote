# 一、HTTPS 是什么？

**HTTPS = HTTP + SSL/TLS**

- 在 HTTP 基础上加入 **加密、身份验证、数据完整性校验**
- 默认端口：**443**
- 传输层仍然是 **TCP**

📌 本质：  
> **HTTP 明文传输 → HTTPS 加密传输**

---

# 二、HTTPS 的三大核心作用（必背）

### ✅ 1️⃣ 加密（Confidentiality）

- 防止数据被窃听
- 使用 **对称加密 + 非对称加密**

📌 典型场景：
- 登录密码
- 支付信息
- 聊天内容

---

### ✅ 2️⃣ 身份验证（Authentication）

- 确认服务器身份
- 防止 **中间人攻击**

📌 依靠：
- **CA 数字证书**
- 服务器公钥

---

### ✅ 3️⃣ 数据完整性（Integrity）

- 防止数据在传输过程中被篡改
- 使用 **消息摘要 / MAC**

📌 确保：
- 数据没被修改
- 数据来源可信

---

# 三、TLS / SSL 是什么？

- **SSL**：早期协议（已淘汰）
- **TLS**：SSL 的升级版（常用 TLS 1.2 / 1.3）

📌 工作位置：
```
应用层（HTTP）
----------------
TLS / SSL
----------------
传输层（TCP）
```

---

# 四、HTTPS 工作流程（简化版）

1. 客户端请求 HTTPS 网站
2. 服务器返回 **数字证书**
3. 客户端验证证书合法性
4. 双方协商 **对称加密密钥**
5. 后续通信全部加密

✅ 特点：
- 握手阶段：非对称加密
- 通信阶段：对称加密
