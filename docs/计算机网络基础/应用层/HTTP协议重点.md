# 一、HTTP 简介（必背）

- HTTP：**超文本传输协议**
- 基于 **TCP**
- 默认端口：**80**
- HTTPS 端口：**443**
- **无状态协议**（每次请求独立）
- 采用 **请求 / 响应模型**

---

# 二、HTTP 请求方法（重点）

| 方法 | 作用 | 是否幂等 | 常见用途 |
|----|----|----|----|
| GET | 获取资源 | ✅ | 浏览网页、查询数据 |
| POST | 提交数据 | ❌ | 表单提交、登录 |
| PUT | 创建 / 更新资源 | ✅ | 修改用户信息 |
| DELETE | 删除资源 | ✅ | 删除订单、用户 |

### ✅ 高频考点
- **GET**
  - 参数在 URL 中
  - 有长度限制
  - 可被缓存、收藏
- **POST**
  - 参数在请求体中
  - 更安全
  - 用于提交敏感数据

📌 **幂等性**：多次执行结果是否一致  
- GET / PUT / DELETE：幂等  
- POST：不幂等

# 三、HTTP 状态码（非常重要）

### 1️⃣ 1xx（信息类）
- **100 Continue**：继续发送请求体

### 2️⃣ 2xx（成功）
- **200 OK**：请求成功 ✅
- **201 Created**：资源创建成功（PUT/POST）

### 3️⃣ 3xx（重定向）
- **301 Moved Permanently**：永久重定向
- **302 Found**：临时重定向
- **304 Not Modified**：资源未修改（缓存）

### 4️⃣ 4xx（客户端错误）
- **400 Bad Request**：请求错误
- **401 Unauthorized**：未认证
- **403 Forbidden**：无权限
- **404 Not Found**：资源不存在 ❌

### 5️⃣ 5xx（服务器错误）
- **500 Internal Server Error**：服务器异常
- **502 Bad Gateway**：网关错误
- **503 Service Unavailable**：服务不可用

✅ **考试常考**
- 200 ✅
- 301 / 302
- 404 ❌
- 500 ❌

# 四、HTTP 请求头 & 响应头（重点）

## 1️⃣ 请求头（Request Header）

常见字段：
- `Host`：目标主机
- `User-Agent`：浏览器信息
- `Content-Type`：请求体类型
- `Cookie`：客户端身份标识
- `Authorization`：身份认证

---

## 2️⃣ 响应头（Response Header）

常见字段：
- `Content-Type`：返回数据类型
- `Set-Cookie`：服务器下发 Cookie
- `Location`：重定向地址（3xx）
- `Cache-Control`：缓存策略

---

# 五、Cookie 与 Session（必考）

## 1️⃣ Cookie

- 存储在 **客户端（浏览器）**
- 每次请求自动携带
- 用于：
  - 登录状态
  - 购物车
  - 用户偏好

✅ 特点：
- 不安全（可被篡改）
- 有大小和数量限制

---

## 2️⃣ Session

- 存储在 **服务器端**
- Session ID 通过 Cookie 传递
- 用于：
  - 用户登录状态
  - 权限控制

✅ 特点：
- 更安全
- 占用服务器资源

---

# 六、Cookie vs Session（面试高频）

| 对比项 | Cookie | Session |
|----|----|----|
| 存储位置 | 客户端 | 服务器 |
| 安全性 | 较低 | 较高 |
| 容量 | 小 | 大 |
| 传输方式 | 每次请求携带 | SessionID |

---

# 七、HTTP 无状态 & 解决方案

❓ 问题：
- HTTP 本身**不记录用户状态**

✅ 解决方案：
- Cookie
- Session
- Token（JWT）

---

# 八、一句话速记（背诵版）

> HTTP 是 TCP 上的请求/响应协议，  
> GET 查、POST 提交、PUT 改、DELETE 删，  
> 2xx 成功、4xx 客户端错、5xx 服务器错，  
> Cookie 存在浏览器，Session 存在服务器。
