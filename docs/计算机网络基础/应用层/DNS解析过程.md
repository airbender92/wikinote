# 一、什么是 DNS？

**DNS（Domain Name System）**  
👉 把**域名解析成 IP 地址**的系统

📌 举例：
```
www.baidu.com → 110.242.68.66
```

- 协议：**UDP（主要） / TCP**
- 端口：**53**

---

# 二、DNS 解析流程（完整版）

> **浏览器缓存 → 系统缓存 → 路由器缓存 → 递归 DNS → 根域名服务器**

---

## ① 浏览器缓存（最快）

- 浏览器会缓存最近访问过的域名解析结果
- 若命中，直接返回 IP

✅ 优点：
- 速度快
- 减少网络请求

---

## ② 系统缓存（hosts + DNS Cache）

- 查询：
  - `hosts` 文件
  - 操作系统 DNS 缓存

📌 路径示例：
```
C:\Windows\System32\drivers\etc\hosts
```

---

## ③ 路由器缓存

- 路由器可能缓存 DNS 结果
- 常见于家庭网络

✅ 作用：
- 减少外部 DNS 查询

---

## ④ 递归 DNS 服务器（运营商 / 公共 DNS）

- 如：
  - 运营商 DNS
  - 8.8.8.8（Google）
  - 1.1.1.1（Cloudflare）

📌 作用：
- 帮用户完成**完整解析流程**
- 缓存解析结果

---

## ⑤ 根域名服务器（Root DNS）

- 全球共 13 组
- 不直接返回 IP
- 返回 **顶级域名服务器地址**

---

## ⑥ 顶级域名服务器（TLD）

- 管理 `.com`、`.cn`、`.org`
- 返回 **权威 DNS 服务器地址**

---

## ⑦ 权威域名服务器（Authoritative DNS）

- 保存域名与 IP 的真实对应关系
- 返回 **最终 IP 地址**

---

## ⑧ 结果返回并缓存

- IP 逐级返回
- 各层 DNS 服务器缓存结果
- 浏览器发起 HTTP / HTTPS 请求

---

# 四、DNS 优化机制

✅ 缓存机制（所有层级）  
✅ TTL（生存时间）  
✅ CDN（就近解析）

---

# 五、一句话必背（考试 / 面试）

> **DNS 解析过程从浏览器缓存开始，依次经过系统、路由器、递归 DNS，最终查询根域名服务器完成域名到 IP 的解析。**
