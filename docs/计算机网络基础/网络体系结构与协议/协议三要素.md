# 网络体系结构与协议

## 1. 什么是网络协议
为进行网络中的数据交换而**建立的规则、标准或约定**，叫做**网络协议（Protocol）**。

通俗说：
**计算机之间交流必须遵守的“语言规则”。**

---

# 2. 协议三要素（必考）
语法、语义、时序（同步）

## ① 语法（Syntax）
**定义：规定数据“长什么样”**
- 数据的格式、结构
- 先传什么、后传什么
- 编码方式、电平高低等

一句话记忆：
**语法 = 长得什么样（格式/结构）**

举例：
HTTP 请求头格式、IP 报文结构、帧格式。

---

## ② 语义（Semantics）
**定义：规定“每个部分是什么意思”**
- 某个比特/字段代表什么含义
- 什么是正确、什么是错误
- 出现某种情况该做什么动作

一句话记忆：
**语义 = 表达什么含义（意义/动作）**

举例：
- TCP 的 SYN 表示“请求建立连接”
- HTTP 200 表示“成功”
- 校验和错误表示“数据包损坏”

---

## ③ 时序（Timing / 同步）
**定义：规定“什么时候发、按什么顺序发”**
- 事件执行顺序
- 发送速度匹配
- 应答等待时间

一句话记忆：
**时序 = 什么时候做、按什么顺序做**

举例：
- TCP 三次握手的顺序
- 先发请求，再等响应
- 超时重传的时间控制

---

# 3. 终极记忆口诀（必背）
- **语法：怎么说（格式）**
- **语义：说什么（含义）**
- **时序：什么时候说、按什么顺序说**

---

# 4. 生活类比（秒懂）
以打电话为例：
- **语法**：必须说汉语/英语，声音要清晰（格式）
- **语义**：“喂”代表打招呼，“再见”代表结束（含义）
- **时序**：先拨号 → 对方接听 → 再说话（顺序）
