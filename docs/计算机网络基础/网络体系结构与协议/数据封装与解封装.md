# 数据封装与解封装（超清晰版 · 必背）
一句话核心：
**从上到下封装（逐层加头），从下到上解封装（逐层去头）**

---

## 1. 什么是封装（Encapsulation）
**发送数据时：从上到下，每层加一个头部**

1. 应用层 → 生成**数据**
2. 传输层 → 加上 **TCP 头** → 成为 **报文段**
3. 网际层 → 加上 **IP 头** → 成为 **IP 数据报**
4. 网络接口层 → 加上 **MAC 帧头 + 帧尾** → 成为 **帧**
5. 最后变成比特流 01 从物理层发出去

就像：
写信 → 装信封 → 写地址 → 贴快递单 → 运输

---

## 2. 什么是解封装（Decapsulation）
**接收数据时：从下到上，每层去掉对应头部**

1. 物理层收到 01 比特
2. 网络接口层 → 去掉帧头帧尾 → 上交网络层
3. 网际层 → 去掉 IP 头 → 上交传输层
4. 传输层 → 去掉 TCP 头 → 上交应用层
5. 应用层拿到原始数据

就像：
收到快递 → 拆包装 → 拆信封 → 读到信

---

## 3. 完整流程（一句话记住）
- **发送方：自上而下，层层加头（封装）**
- **接收方：自下而上，层层去头（解封装）**

---

## 4. 每层数据单位名称（必考）
- 应用层：**数据**
- 传输层：**报文段**（TCP）/ 用户数据报（UDP）
- 网络层：**IP 数据报（分组）**
- 数据链路层：**帧**
- 物理层：**比特**

记忆口诀：
**数 → 段 → 报 → 帧 → 比特**
