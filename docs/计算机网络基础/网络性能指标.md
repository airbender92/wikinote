# 网络性能指标（详细讲解）

## 1. 速率（Rate）
**定义**：
单位时间内传输的**二进制比特数**，单位是 **bit/s（bps）**。

- 1 kbps = 1000 bps
- 1 Mbps = 1000 kbps
- 1 Gbps = 1000 Mbps

**通俗理解**：
网络的**“最快理论速度”**。

**举例**：
百兆网 100Mbps，千兆网 1000Mbps。

---

## 2. 带宽（Bandwidth）
**定义**：
信道能传输信号的**频率范围**，用来表示网络的**最大传输能力**。
单位同样是 **bps**。

**通俗理解**：
马路的**宽度**。
路越宽，能同时跑的车越多 → 网速越快。

**注意**：
日常说的“带宽100M”，实际指的就是**最高速率 100Mbps**。

---

## 3. 时延（Latency / Delay）
**定义**：
数据从发送方到接收方**总共花费的时间**。
单位：ms（毫秒）。

时延由四部分组成：

### ① 发送时延（传输时延）
数据从主机“推”到链路上花的时间。
公式：
> 发送时延 = 数据长度 / 发送速率

**例子**：
把一个大文件从电脑发到网卡的时间。

### ② 传播时延
信号在物理线路上跑的时间。
公式：
> 传播时延 = 信道长度 / 电磁波传播速度

**例子**：
光纤里光速传播，距离越远，时延越大。

### ③ 处理时延
路由器/交换机解析、检查分组花费的时间。

### ④ 排队时延
数据包在路由器队列里等待转发的时间。
拥堵越严重，排队越久。

**总时延**
> 总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延

---

## 4. 吞吐量（Throughput）
**定义**：
单位时间内**实际成功传输**的数据量。
单位：bps、Mbps。

**通俗理解**：
**实际跑起来的速度**，不是理论值。

**举例**：
带宽 100M，但网络拥堵，实际吞吐量只有 20Mbps。

**和带宽的区别**：
- 带宽 = **最大能力**
- 吞吐量 = **实际表现**

---

## 5. 丢包率（Packet Loss Rate）
**定义**：
传输中**丢失的数据包**占总发送数据包的比例。

公式：
> 丢包率 = 丢失包数 / 总发送包数 × 100%

**影响**：
- 丢包高 → 看视频卡顿、游戏延迟跳ping、网页加载慢
- TCP 会重传，导致更慢

**正常标准**：
一般网络丢包率 < 1% 算良好。

---

# 极简总结（适合背诵）
- **速率**：理论最高速度（bps）
- **带宽**：信道容量，马路宽度
- **时延**：数据跑一趟的总时间（发送+传播+处理+排队）
- **吞吐量**：实际传输速度
- **丢包率**：丢包比例，越低越好
