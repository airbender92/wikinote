### 局域网技术：以太网（Ethernet）与CSMA/CD  


#### 一、以太网（Ethernet）：最主流的局域网技术  
以太网（Ethernet）是**应用最广泛的局域网（LAN）技术标准**，由Xerox、Intel、DEC于1973年联合开发，后由IEEE标准化为**IEEE 802.3系列协议**。它定义了局域网的**物理层（PHY）** 和**数据链路层（MAC子层）** 规范，核心目标是实现**相邻节点间的可靠、高速数据传输**。  


##### 1. 以太网的核心特点  
- **拓扑结构**：早期为**总线型**（所有节点共享一根同轴电缆），现代为**星型**（通过交换机/集线器连接，交换机为核心）；  
- **传输介质**：双绞线（UTP/STP，如Cat5e/Cat6，最常用）、光纤（单模/多模，长距离高速）、同轴电缆（早期，已淘汰）；  
- **速率演进**：从10Mbps（10BASE-T）→ 100Mbps（Fast Ethernet）→ 1Gbps（Gigabit Ethernet）→ 10Gbps/40Gbps/100Gbps（万兆/十万兆以太网），兼容向下；  
- **帧结构**：固定格式的“以太网帧”（见下文），含MAC地址、数据、校验码（CRC）；  
- **介质访问控制**：采用**CSMA/CD协议**（半双工模式）或**全双工交换**（现代主流，无冲突）。  


##### 2. 以太网的发展与标准化  
- **标准演进**：IEEE 802.3定义了基带传输（数字信号直接传输），细分不同介质和速率：  
  - 10BASE-T：双绞线，10Mbps，星型拓扑（用集线器）；  
  - 100BASE-TX：双绞线，100Mbps（Fast Ethernet）；  
  - 1000BASE-T：双绞线，1Gbps（Gigabit Ethernet，用4对线）；  
  - 10GBASE-SR：光纤，10Gbps（短距离，多模光纤）。  
- **交换式以太网**：1990年代后，交换机取代集线器，每个端口独立带宽（全双工），**分割冲突域**（每个端口为一个冲突域），大幅提升性能。  


##### 3. 以太网帧结构（IEEE 802.3）  
以太网帧是数据链路层的传输单位，结构如下（括号内为字节数）：  
```  
[前导码(7B)][帧起始定界符(1B)][目的MAC(6B)][源MAC(6B)][类型/长度(2B)][数据(46~1500B)][FCS(4B, CRC校验码)]  
```  
- **前导码**：7字节“10101010”，用于接收方时钟同步；  
- **帧起始定界符（SFD）**：1字节“10101011”，标识帧开始；  
- **目的/源MAC地址**：6字节全球唯一物理地址（如00:1A:2B:3C:4D:5E），由IEEE分配；  
- **类型/长度**：2字节，标识上层协议（如0x0800=IP，0x0806=ARP）或数据长度；  
- **数据**：46~1500字节（不足46字节需填充），承载网络层分组（如IP数据报）；  
- **FCS**：4字节CRC校验码，用于差错检测（接收方校验错误则丢弃帧）。  


#### 二、CSMA/CD：以太网的介质访问控制协议  
CSMA/CD（Carrier Sense Multiple Access with Collision Detection，载波监听多路访问/冲突检测）是**传统共享式以太网（半双工模式）的核心MAC协议**，用于解决**多节点共享同一传输介质时的冲突问题**（如早期总线型以太网、集线器连接的星型网）。  


##### 1. 核心问题：共享介质的冲突  
当多个节点同时通过总线发送数据时，信号会叠加导致**冲突**（Collision），数据失真。CSMA/CD的目标是：**让节点“有序竞争”发送权，冲突后高效恢复**。  


##### 2. CSMA/CD的三大机制  
CSMA/CD基于“**先听后说、边说边听、冲突后退**”的原则，分三步：  

###### （1）载波监听（Carrier Sense）：先听后说  
- **发送前监听**：节点发送数据前，先检测信道是否空闲（有无其他节点发送信号）。  
  - 若信道空闲（无载波），则开始发送；  
  - 若信道忙（有载波），则继续监听，直到空闲后延迟一小段时间（避免多个节点同时抢发）再发送。  

###### （2）多点接入（Multiple Access）：多节点共享介质  
- 所有节点连接到同一传输介质（如总线、集线器），平等竞争发送权（“多点接入”）。  

###### （3）冲突检测（Collision Detection）：边说边听  
- **发送中检测**：节点发送数据时，持续监听信道，比较“自己发送的信号”与“信道实际信号”。  
  - 若一致：无冲突，继续发送；  
  - 若不一致（如信号幅度异常、波形畸变）：检测到冲突，立即停止发送，并发送**干扰信号（Jam Signal，32位全1）** 通知全网。  


##### 3. 冲突处理：二进制指数退避算法  
冲突发生后，节点需等待随机时间后重试，避免再次冲突。具体用**二进制指数退避算法**：  
1. **设定退避次数**：首次冲突后，退避时间从{0,1}中随机选一个数（乘以基本退避时间，如51.2μs）；  
2. **二次冲突后**：退避时间从{0,1,2,3}中随机选；  
3. **三次冲突后**：从{0~7}中选；  
4. **最多16次冲突**：若16次仍冲突，放弃发送并报告错误。  

**原理**：冲突次数越多，退避范围越大，降低再次冲突概率（类似“人多抢座，越挤越慢”）。  


##### 4. CSMA/CD的工作流程（示例）  
```  
节点A准备发送数据：  
1. 监听信道→空闲→开始发送；  
2. 发送中持续监听→发现信号异常（与自身发送不符）→检测到冲突；  
3. 立即停止发送，发干扰信号；  
4. 记录冲突次数（首次冲突），从{0,1}中随机选退避时间（如选1×51.2μs=51.2μs）；  
5. 等待51.2μs后，重新监听信道→空闲→再次发送。  
```  


##### 5. CSMA/CD的局限性与现代以太网的改进  
- **局限性**：仅适用于**半双工模式**（同一时间只能发或收），且**冲突域**内节点越多，冲突概率越高（效率越低）。  
- **现代改进**：  
  - **全双工以太网**：交换机为每个端口提供独立发送/接收通道（双工通信），无需共享介质，**彻底消除冲突**，因此不再需要CSMA/CD；  
  - **交换式以太网**：交换机分割冲突域（每个端口一个冲突域），集线器（共享式）已被淘汰，CSMA/CD仅用于早期共享式网络。  


#### 三、以太网与CSMA/CD的关键总结  
| **维度**         | **以太网（Ethernet）**                          | **CSMA/CD协议**                          |  
|------------------|-------------------------------------------------|------------------------------------------|  
| **定位**         | 局域网技术标准（IEEE 802.3），定义物理层+MAC层   | 以太网的MAC子层协议（半双工模式）         |  
| **核心功能**     | 提供帧传输、MAC寻址、速率适配                   | 解决共享介质冲突，实现多节点有序接入     |  
| **关键机制**     | 帧结构（MAC地址+CRC）、速率演进（10M~100Gbps）  | 载波监听、冲突检测、二进制指数退避         |  
| **适用场景**     | 所有局域网（星型/总线型，双绞线/光纤）           | 早期共享式以太网（集线器连接，半双工）     |  
| **现代地位**     | 仍是局域网主流（交换式以太网占绝对主导）         | 半双工模式已淘汰，全双工无需CSMA/CD        |  


#### 四、核心结论  
- **以太网**是局域网的“代名词”，通过标准化的帧结构和不断升级的速率（10M→100Gbps），成为全球部署最广的局域网技术；  
- **CSMA/CD**是以太网发展初期的“功臣”，通过“先听后说、冲突后退”解决了共享介质的冲突问题，但仅适用于半双工共享式网络；  
- **现代以太网**已进入“全双工交换时代”：交换机分割冲突域，节点间独立通信，无需CSMA/CD，性能（吞吐量、延迟）大幅提升。  

理解以太网和CSMA/CD，是掌握局域网技术的基础——前者定义了“怎么传”，后者解决了“如何有序传”的问题。
