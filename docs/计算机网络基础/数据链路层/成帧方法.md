成帧方法（Framing Methods）

成帧是数据链路层的核心功能之一，目的是将网络层的分组（Packet）封装为帧（Frame），通过添加边界标识（帧头/帧尾）解决物理层“比特流无边界”的问题，确保接收方能正确识别“一帧数据的开始与结束”。常见的成帧方法包括字符计数法、首尾字符法（字符填充法）、零比特填充法，以及违例编码法（如以太网）。以下详细解释前三种：  

一、字符计数法（Character Counting）

定义

通过一个计数字段（Count Field） 显式记录帧的长度（字符数），接收方根据该计数值截取完整帧。计数字段通常位于帧头（如1字节或2字节）。  

原理

1. 发送方：将网络层分组封装为帧，在帧头添加一个计数字段，值为帧的总字符数（不含计数字段本身？或含？需协议约定，通常为含）；  
2. 接收方：读取帧头的计数值 N，然后从下一字节开始截取 N 个字符作为一个完整帧，后续字节作为下一帧的开始。  

示例（假设计数字段为1字节，最大值255字符）

  
帧结构：[计数字段(1B)][数据(共N字符)]  
例如：计数字段=5，数据="ABCDE" → 完整帧为 [0x05][A][B][C][D][E]（共6字节，计数字段值5对应数据长度5字符）  
  

优缺点

• 优点：实现简单，直接通过计数值定位帧边界，无需复杂填充规则。  

• 缺点：致命缺陷——若计数字段本身出错（如受噪声干扰从5变为6），接收方会错误地截取后续6个字符作为一帧，导致后续所有帧同步错位（“多米诺骨牌效应”），难以恢复同步。  

典型应用

早期低速异步通信协议，如IBM的BISYNC协议（Binary Synchronous Communications），但因可靠性差，已被淘汰。  

二、首尾字符法（Character Stuffing，字符填充法）

定义

用特殊的起始字符（如SOH，Start of Header） 和结束字符（如EOT，End of Transmission） 标记帧的边界，若数据中出现与起始/结束字符相同的字节，则用转义字符（如ESC，Escape） 填充（插入额外字节），避免接收方误判边界。  

原理

1. 边界标识：帧以SOH（0x01）开头，以EOT（0x04）结尾；  
2. 数据填充规则：  
   • 若数据中某字节等于SOH或EOT，则在该字节前插入转义字符ESC（0x1B）；  

   • 若数据中某字节等于ESC本身，则在ESC前再插入一个ESC（即连续两个ESC），接收方删除多余的ESC；  

3. 接收方处理：遇到SOH开始接收，遇到EOT结束接收；遇到ESC则跳过下一个字节（视为填充的转义标记），恢复原数据。  

示例

• 原始数据（含SOH）：[SOH]Hello[SOH]World[EOT]（假设SOH=0x01，EOT=0x04）  

• 填充后数据（插入ESC）：[SOH][ESC][SOH]Hello[ESC][SOH]World[ESC][EOT][EOT]  

  （注：数据中第一个SOH前无需填充，因帧头已有SOH；数据中SOH/EOT前加ESC，ESC本身也加ESC）  

优缺点

• 优点：直观易懂，用可见字符（ASCII）界定边界，适合文本数据传输。  

• 缺点：  

  • 依赖字符编码：仅适用于字符型数据（如ASCII），若传输二进制数据（含任意字节），可能出现大量填充（如数据中频繁出现SOH/EOT）；  

  • 透明性差：无法实现“完全透明传输”（数据中必须避免出现特殊字符，或通过填充处理，效率低）；  

  • 边界风险：若起始/结束字符（SOH/EOT）出错（如丢失或被干扰），接收方无法识别帧边界。  

典型应用

早期低速协议，如DEC的DDCMP协议（Digital Data Communications Message Protocol）、部分串行通信协议（如早期Modem通信）。  

三、零比特填充法（Zero Bit Stuffing）

定义

用特定的比特模式（如01111110，称为“标志位Flag”） 作为帧头和帧尾，发送方在数据中遇到连续5个“1”时自动插入一个“0”，接收方收到连续5个“1”后删除后续的“0”，恢复原始数据（避免数据中意外出现标志位导致边界误判）。  

原理

1. 边界标识：帧以“01111110”（Flag）开头和结尾；  
2. 发送方填充规则：扫描待发送的比特流，若连续出现5个“1”，则在其后强制插入一个“0”（无论第6位是否为0）；  
3. 接收方去填充规则：扫描接收的比特流，若连续出现5个“1”且其后为“0”，则删除该“0”；若连续出现6个“1”，则为帧边界（Flag）。  

示例

• 原始数据比特流：010111110101...（含连续5个1：“11111”）  

• 发送方填充后：0101111[0]10101...（在连续5个1后插入0，变为“111110”）  

• 接收方去填充后：检测到“111110”，删除0，恢复为“11111”  

优缺点

• 优点：  

  • 透明传输：不依赖字符编码，可传输任意二进制数据（包括连续1的比特流）；  

  • 可靠性高：标志位（01111110）通过“5个1+0”的规则避免与数据混淆，同步恢复能力强；  

  • 主流方法：现代数据链路层协议广泛采用（如HDLC、PPP）。  

• 缺点：需逐比特处理数据（软件实现稍复杂，但硬件易优化）。  

典型应用

• HDLC协议（High-Level Data Link Control，高级数据链路控制）：广泛用于X.25、帧中继等广域网；  

• PPP协议（Point-to-Point Protocol，点到点协议）：用于拨号上网、路由器串口连接；  

• 以太网（早期）：虽以太网用违例编码法（曼彻斯特编码的特殊电平），但零比特填充法是理论上的经典透明传输方案。  

三种成帧方法对比总结

成帧方法 边界标识 核心机制 优点 缺点 典型应用
字符计数法 帧头计数字段（字符数） 计数值=帧长，接收方按长度截取 简单直接 计数字段出错导致同步丢失 IBM BISYNC协议（已淘汰）
首尾字符法 起始/结束字符（如SOH/EOT） 数据中特殊字符用ESC转义填充 直观，适合文本数据 依赖字符编码，透明性差 DEC DDCMP协议（早期低速）
零比特填充法 比特标志位（如01111110） 连续5个1后插0，接收方删0恢复数据 透明传输，适合二进制数据，可靠 需逐比特处理（硬件易优化） HDLC、PPP协议（主流）
  

核心结论

• 字符计数法因同步脆弱性已被淘汰；  

• 首尾字符法（字符填充）仅用于早期简单文本通信；  

• 零比特填充法因透明性和可靠性成为现代数据链路层的主流成帧方法（如PPP、HDLC），而以太网则用“违例编码法”（曼彻斯特编码的特殊电平组合）作为补充。
