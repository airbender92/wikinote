# Problem: You need to serve static content with NGINX.

# Solution: 

**把 NGINX 默认的网站配置文件（`/etc/nginx/conf.d/default.conf`）清空，替换成我给你的这段新配置。**
```nginx
server { 
  listen 80 default_server; 
  server_name www.example.com;
  location / { 
    root /usr/share/nginx/html; 
    # alias /usr/share/nginx/html;
    index index.html index.htm;
  }
}
```

这段配置的作用：
- 监听 **80 端口**（网页默认端口）
- 设置网站域名：`www.example.com`
- 指定网站文件存放目录：`/usr/share/nginx/html`
- 默认打开首页：`index.html`

---

# 对你（Docker 环境）的操作步骤
你只需要做 2 件事：
1. 打开这个默认配置文件
2. 把里面内容**全删掉**，粘贴这段新代码

## 命令（直接照着输）
```bash
# 编辑默认配置文件
vi /etc/nginx/conf.d/default.conf
```

## 编辑方法：
1. 按 `i` 进入编辑模式
2. **把原有内容全部删除**
3. 粘贴这段代码：
```nginx
server {
  listen 80 default_server;
  server_name www.example.com;

  location / {
    root /usr/share/nginx/html;
    index index.html index.htm;
  }
}
```
4. 按 `Esc`，输入 `:wq` 保存退出

## 最后让配置生效
```bash
nginx -t && nginx -s reload
```

# 整体大白话解释
这段配置是 Nginx 最基础的**静态网站配置**，作用是：
在 80 端口提供网页服务，网页文件放在 `/usr/share/nginx/html`。

- `server { ... }` 表示**一个网站**。
- `listen 80 default_server` 表示监听 80 端口，并且是**默认网站**——如果没有匹配到其他网站，就用这个。
- `server_name` 是域名，访问时带上这个域名才会进入这个网站；但因为设了 `default_server`，就算不写域名也能访问。
- `location /` 匹配**所有 URL 请求**。
- `root` 指定网站文件所在目录，Nginx 会根据访问的路径去这个目录找对应的文件。
- `index` 表示当访问路径没有指定具体文件时，默认去找 `index.html` 这类首页文件。

---

# 超精简总结
这就是一个**最基础、最标准的静态网站配置**：
监听 80 端口 → 匹配所有请求 → 从指定目录读取静态文件 → 自动找首页。
