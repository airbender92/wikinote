# 7.6 保护一个目录（Secure Link）
## **超级通俗、彻底讲懂**

我给你用人话讲，**保证你一看就会、马上能用**👇

---

# 核心一句话
**secure_link = 给下载链接加一个临时、加密的“通行证”
没有正确加密串 → 直接 403 禁止
有 → 才能下载
防止别人盗链、防止随便访问你的私密文件**

---

# 这个功能是干嘛的？
比如你有这些资源：
- 私密下载文件
- 视频
- 图片
- 付费资源

你**不想让别人直接输入网址就能访问**，
必须通过你生成的**带加密串的合法链接**才能访问。

---

# 你给的配置逐行解释
```nginx
location /resources {
    secure_link_secret mySecret;  # 加密密钥（只有你知道）

    if ($secure_link = "") {     # 如果验证失败
        return 403;              # 直接禁止
    }

    rewrite ^ /secured/$secure_link;  # 验证通过 → 内部转发
}

location /secured/ {
    internal;  # 标记为内部位置，外部无法直接访问
    root /var/www;
}
```

---

# 它是怎么工作的？（超级简单）

## 1）你后端生成一个**带 MD5 签名的地址**
规则：
```
md5( 密钥 + 文件路径 )
```

例子：
- 密钥：`mySecret`
- 文件：`video.mp4`
- 加密串：`md5("mySecretvideo.mp4")`

生成的地址：
```
https://your.com/resources/【加密串】/video.mp4
```

## 2）用户访问这个地址
## 3）Nginx 用密钥重新计算 MD5
## 4）和你给的加密串对比
- ✅ 一样 → 允许访问
- ❌ 不一样 → 403 禁止

## 5）最后内部转发到 `/secured/` 目录
外部**永远无法直接访问** `/secured/`

---

# 最关键的点
### **secure_link_secret 是你的密钥**
- 存在 Nginx 里
- 不能泄露
- 必须是固定字符串，**不能是变量**

### **$secure_link**
- 验证通过 → 存放真实文件名
- 验证失败 → 空字符串

---

# 这个配置的作用总结成 3 句
1. **外部只能访问 /resources**
2. **必须带正确的加密签名**
3. **真实文件藏在 /secured/，外部绝对访问不到**

---

# 超精简总结
**secure_link = 给文件链接加密码
密钥存在 Nginx
链接必须带正确 MD5 串
否则一律 403
真实文件藏在内部目录，绝对安全**
