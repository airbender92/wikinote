# 安全控制（Security Controls）· 章节概览
这段是**第7章（安全控制）的引言**，我给你用最直白、不啰嗦的方式翻译+总结👇

---

## 核心意思
**安全要靠多层防御（深度防御），
只靠一层防护是不够的，必须多层叠加才叫真正加固。**

本章会教你用 NGINX 实现各种**Web应用安全加固手段**，
并且很多方法可以**组合使用**，让系统更安全。

---

## 里面提到的重点
1. **本章不讲 ModSecurity 3.0**
   也就是那个能把 Nginx 变成 **WAF（Web应用防火墙）** 的模块。
   想了解 WAF 要去看官方的《Quick Start Guide》。

2. **重要公告**
   - **NGINX Plus 的 ModSecurity WAF 已经在 2024-03-31 进入生命周期结束（EoL）**
   - 官方不再维护这个模块了
   - 想了解详情去看 F5 官方博客

---

## 本章大概会讲哪些内容（按同类书籍惯例）
你后面会看到的典型安全配置包括：
- 禁用不必要的 HTTP 方法（PUT、DELETE 等）
- 限制请求体大小，防上传攻击
- 设置安全头部（`X-Frame-Options`、`X-XSS-Protection`、`HSTS`、CSP 等）
- 屏蔽敏感信息（隐藏 Nginx 版本号）
- 访问控制（允许/禁止 IP）
- 防 SQL 注入、XSS 的基础规则
- HTTPS 强制、SSL/TLS 加固
- 限流、防爬虫、防 CC（前面章节已经讲过）

一句话概括：
**这一章 = 用 Nginx 做“前置安全守门员”，在入口层就挡住大量常见攻击。**
