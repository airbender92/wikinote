# 7.14 **`satisfy` 指令：多条件安全验证（满足任意一个 / 全部）**
我给你用人话讲得**清清楚楚、一看就懂**👇

---

# 核心一句话
**`satisfy any` = 满足**任意一个**安全条件就放行
`satisfy all` = 必须满足**所有**安全条件才放行**

这是 Nginx 最强大的**多层安全验证**规则。

---

# 先看你给的配置
```nginx
location / {
    satisfy any;       # 满足任意一个条件即可

    allow 192.168.1.0/24;  # 条件1：内网IP
    deny all;              # 其他IP拒绝

    auth_basic "closed site";          # 条件2：账号密码登录
    auth_basic_user_file conf/htpasswd;
}
```

---

# 它到底实现了什么？
## 满足 **任意一个** 条件就能访问：
1. **来自内网 192.168.1.0/24** → 直接进
**或者**
2. **输入正确账号密码** → 也能进

---

# 两种模式的区别（超级重要）

## ✅ `satisfy any;`
**满足一个就放行**
- IP 允许 → 进
- 或 密码正确 → 进

## ✅ `satisfy all;`
**必须全部满足**
- IP 必须允许
**并且**
- 密码必须正确

---

# 这个配置的真实场景
你想做一个**内部站点**：
- **公司内网用户 → 直接访问，不用输密码**
- **外网用户 → 必须输账号密码才能进**

→ 这就是 `satisfy any` 的经典用法！

---

# 可以和哪些安全规则一起用？
- IP 白名单/黑名单 (`allow/deny`)
- 基础账号密码认证 (`auth_basic`)
- 第三方认证服务 (`auth_request`)
- JWT 认证

只要是安全类规则，`satisfy` 都能组合。

---

# 超精简总结
```
satisfy any   = 满足一个条件就过
satisfy all   = 所有条件都要满足
```

你给的例子：
**内网IP 免密访问
外网需要账号密码**
