# 7.3 客户端加密（SSL/TLS / HTTPS）
## **超通俗、彻底讲懂**

我用最简单、最实用、不绕弯的方式给你讲👇

# 核心一句话
**HTTPS = 在 Nginx 上配置 SSL 证书，让浏览器和服务器之间的传输加密。
别人抓包看不到你的数据、密码、token。**

---

# 1. 这段配置是干嘛的？
```nginx
server {
    listen 8443 ssl;  # 监听 8443 端口，开启 SSL

    ssl_certificate /etc/nginx/ssl/example.crt;    # 公钥证书
    ssl_certificate_key /etc/nginx/ssl/example.key; # 私钥
}
```

作用：
- 开启 **HTTPS 加密服务**
- 端口 `8443`（可以是 443）
- 使用证书文件进行加密传输

---

# 2. 两个文件是什么？
### **ssl_certificate → 公钥证书（给浏览器的）**
浏览器会验证这个证书是否合法。

### **ssl_certificate_key → 私钥（服务器自己留着）**
用来解密请求、加密响应。

**必须配对使用。**

---

# 3. 为什么要加密？
- 防止抓包看到密码、token、隐私数据
- 防止篡改页面内容
- 防止中间人劫持
- 浏览器强制要求 HTTPS 才能用很多功能（摄像头、定位、蓝牙等）

---

# 4. SSL vs TLS
- **SSL 老协议，不安全**
- **TLS 是现代安全版本**
- 现在用的都是 **TLS 1.2 / 1.3**

Nginx 会自动使用安全的版本。

---

# 5. 证书从哪来？
- 免费：**Let’s Encrypt**
- 收费：阿里云、腾讯云、GeoTrust、DigiCert
- 测试：自己生成自签名证书

---

# 6. 最标准、最完整的 HTTPS 配置（直接复制用）
```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /etc/nginx/ssl/your.crt;
    ssl_certificate_key /etc/nginx/ssl/your.key;

    # 安全加密配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
}

# 把 HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$host$request_uri;
}
```

---

# 7. 一句话总结本章
**HTTPS = 配置 SSL 证书
加密浏览器 ↔ Nginx 之间的传输
保护隐私、防劫持、更安全**
