# 7.7 用密钥生成安全链接（超级通俗版）
这一节就是教你：**后端代码怎么生成符合 Nginx 规则的加密链接**，和上一节 **7.6 配对使用**。

我用最简单、最直白、一看就懂的方式讲👇

---

# 核心一句话
**加密链接 = 文件路径 + 密钥 → 拼在一起 → MD5 计算 → 放到 URL 里**

Nginx 收到后，用同样的方式算一遍，对得上就放行。

---

# 1. 加密规则（死记这一条）
```
md5( 文件路径 + 密钥 )
```

注意：
- **先写文件名**
- **再写密钥**
- **中间什么都不加**
- **不能有换行**

---

# 2. 看例子（你给的）
文件：`index.html`
密钥：`mySecret`

拼接字符串：
```
index.htmlmySecret
```

计算 MD5：
```
a53bee08a4bf0bbea978ddf73636a12
```

---

# 3. 最终可访问的 URL 格式
```
http://example.com/resources/【MD5值】/【文件名】
```

真实可用地址：
```
www.example.com/resources/a53bee08a4bf0bbea978ddf736363a12/index.html
```

---

# 4. 命令行生成方式（你给的）
```bash
echo -n 'index.htmlmySecret' | openssl md5 -hex
```
输出就是 MD5 值。

---

# 5. 代码生成方式（Python）
```python
import hashlib
hashlib.md5(b"index.htmlmySecret").hexdigest()
```

---

# 6. 整个流程总结（超级清晰）
1. 你在 Nginx 配置密钥：`mySecret`
2. 后端拼接：`文件名 + 密钥`
3. 计算 MD5
4. 拼出 URL：`/resources/MD5值/文件名`
5. 用户访问这个 URL
6. Nginx 自己计算一遍 MD5
7. 一致 → 允许访问
8. 不一致 → 403 拒绝

---

# 7. 必须记住的 3 个要点
1. **文件名在前，密钥在后**
2. **拼接不能加任何东西**
3. **用 MD5 十六进制小写**

---

# 超精简总结
**这一节就是教你后端怎么生成 Nginx 认识的加密链接。
公式：md5(文件路径+密钥)
生成后拼在 URL 里即可。**
