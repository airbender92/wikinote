# 7.5 上游加密（Upstream Encryption）
## **超级通俗、彻底讲懂**

我用最简单、最实用、不绕弯的方式给你讲👇

---

# 核心一句话
**前面讲的是：浏览器 ↔ NGINX 加密（HTTPS）
这一节讲的是：NGINX ↔ 后端服务（上游）加密
也就是：Nginx 调用后端接口时，也走 HTTPS，并且验证证书。**

---

# 你必须先搞懂架构
```
用户浏览器 → NGINX → 后端服务（API/Java/PHP/Node.js）
```

前面讲的是**第一段加密**：
```
浏览器 ↔ NGINX（HTTPS）
```

这一节讲**第二段加密**：
```
NGINX ↔ 后端服务（HTTPS）
```

---

# 为什么要给 Nginx → 后端也加密？
因为：
1. **后端在公网上**（不是你的内网）
2. **符合等保/合规要求**（必须全程加密）
3. 防止内网被抓包、窃听
4. 防止中间人篡改

---

# 配置代码解释（你给的）
```nginx
location / {
    proxy_ssl_verify on;          # 开启验证后端证书
    proxy_ssl_verify_depth 2;      # 验证证书链深度
    proxy_pass https://upstream.example.com;  # 重点：https://
}

proxy_ssl_protocols TLSv1.3;      # 只使用 TLS 1.3
```

---

# 逐行翻译人话版

### 1. proxy_pass https://...
→ **Nginx 访问后端用 HTTPS，不是 HTTP**
这是开启加密的关键。

### 2. proxy_ssl_verify on
→ **Nginx 会验证后端的证书是否合法**
（默认是 off，不验证，不安全）

### 3. proxy_ssl_verify_depth 2
→ 验证证书链层级（默认即可）

### 4. proxy_ssl_protocols TLSv1.3
→ 只使用最强加密 TLS 1.3

---

# 最重要的区别（99%的人不懂）
## 不加验证：
```
proxy_pass https://api;
```
→ 加密了，但**不验证证书是否合法**
→ 自签名、过期、伪造证书都能过
→ **不安全**

## 加验证：
```
proxy_ssl_verify on;
proxy_pass https://api;
```
→ 加密 + 严格验证证书
→ **安全、合规**
