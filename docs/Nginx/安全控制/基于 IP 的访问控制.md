# 7.1 基于 IP 的访问控制（超级通俗讲解）
我直接给你**讲透、讲简单、讲实用**，保证你马上能用👇

# 核心一句话
**allow = 允许某个IP/网段访问
deny = 拒绝某个IP/网段访问
规则**从上到下匹配**，匹配到就立刻生效，不再往下走。**

---

# 1. 直接看你给的配置（逐行翻译）
```nginx
location /admin/ {
    allow 10.0.0.0/20;    # 允许这个内网段
    deny 10.0.0.1;        # 但禁止这个具体IP（即使在上面的段里）
    allow 2001:0db8::/32; # 允许IPv6网段
    deny all;             # 其他所有人全部禁止
}
```

### 执行顺序（超级重要）
1. 先看是不是 `10.0.0.0/20` → 是就允许
2. 再看是不是 `10.0.0.1` → 是就拒绝
3. 再看是不是那个 IPv6 → 是就允许
4. **最后 deny all → 全部拒绝**

### 结论：
**只有前面允许的IP能访问，其他人一律403禁止。**

---

# 2. 最关键规则：顺序决定一切
NGINX 的 IP 控制是：
## **从上往下匹配，匹配第一条符合的就停止，不继续检查。**

例子：
```
deny all;
allow 1.1.1.1;
```
→ **全部拒绝**，因为第一条就匹配了。

正确写法：
```
allow 1.1.1.1;
deny all;
```

---

# 3. 支持哪些格式？
- 单个 IP：`allow 192.168.1.1;`
- CIDR 网段：`allow 192.168.1.0/24;`
- IPv6：`allow 2001:db8::1;`
- 所有：`allow all;` 或 `deny all;`

---

# 4. 最常用的生产配置（直接复制）
只允许内网访问后台：
```nginx
location /admin/ {
    allow 127.0.0.1;
    allow 10.0.0.0/8;
    allow 192.168.0.0/16;
    allow 172.16.0.0/12;
    deny all;
}
```

---

# 5. 拒绝后返回什么？
- 默认返回 **403 Forbidden**

---

# 6. 可以用在哪里？
- `http` 全局
- `server` 某个网站
- `location` 某个目录/接口
- `stream` TCP/UDP 代理（如MySQL、Redis）

---

# 超精简总结（永久记住）
**allow = 允许
deny = 拒绝
规则从上到下匹配
最后必须加 deny all
只让信任的IP进，其他人全部拦死**
