# 6.1 HTTP Basic Authentication
## 超通俗完整版讲解

# 核心一句话
**HTTP Basic 认证 = 浏览器弹出账号密码框，验证通过才能访问
最简单的轻量保护，适合内网、测试站、私密目录**

---

# 1. 密码文件格式
文件里一行一个用户，用**冒号分隔**：
```
用户名:加密后的密码
用户名:密码:备注
```

示例：
```
user1:$1$xyz$abcdefghijk
admin:$apr1$xxx$yyy:Admin account
```

---

# 2. 生成加密密码（不要明文！）
## 用 openssl 生成
```bash
openssl passwd 你的密码
```
输出一串加密字符串，填到密码文件里。

## 或者用 htpasswd（更常用）
```bash
htpasswd -c ./passwd 用户名
```
直接生成标准的 Nginx 密码文件。

---

# 3. Nginx 配置
```nginx
location / {
    auth_basic "Private site";        # 弹窗提示语
    auth_basic_user_file conf.d/passwd;  # 密码文件路径
}
```

- `auth_basic`：开启认证，字符串是浏览器弹窗上显示的提示文字
- `auth_basic_user_file`：指定刚才创建的账号密码文件

---

# 4. 浏览器表现
访问时弹出**登录框**：
- 输入正确 → 正常访问
- 输错 3 次 → 401 Forbidden

---

# 5. 用 curl 测试
```bash
curl -u 用户名:密码 http://your.site
```

---

# 6. 底层原理（简单了解）
1. 没登录时 Nginx 返回：
   ```
   401 Unauthorized
   WWW-Authenticate: Basic realm="Private site"
   ```
2. 浏览器弹出输入框
3. 浏览器把 `user:pass` 做 **Base64 编码**
4. 放在请求头里：
   ```
   Authorization: Basic dXNlcjE6cGFzc3dvcmQ=
   ```
5. Nginx 解码后核对密码文件

---

# 7. 超级重要警告
### **Base64 不是加密！只是编码！
必须配合 HTTPS 使用，否则账号密码明文裸奔，抓包就被盗。**

---

# 8. 适用场景
- 测试环境
- 内部文档
- 小范围私密页面
- 简单防止爬虫/陌生人访问

**不适合**：正式用户系统、登录态、高安全场景。

---

# 极简总结
- `auth_basic`：开启弹窗认证
- `auth_basic_user_file`：指定账号密码文件
- 密码必须加密，不能明文
- 必须走 HTTPS
- 最简单轻量的认证方式
