# 认证（Authentication）· 概览白话翻译

这一小段是**本章引言**，我给你翻成完全直白、好懂的版本：

---

NGINX 可以对客户端做**身份认证**。
把认证逻辑放在 NGINX 上，有两大好处：
1. **减轻后端应用服务器压力**
2. **在最外层就拦住未登录/未授权的请求**，不让它们进到应用里

NGINX 开源版提供的认证能力：
- **HTTP Basic Auth**（基础账号密码认证）
- **子请求认证（auth_request）**
  把鉴权交给你自己的后端接口去判断

NGINX Plus 商业版独有：
- **JWT 验证**
  可以直接对接 OpenID Connect 等第三方登录体系（OAuth、SSO 等）

**本章会讲解各种用 NGINX 做认证、保护接口的方法。**

---

### 简单总结
- **Basic Auth**：最简单的账号密码登录
- **auth_request**：最灵活，调用你自己的接口鉴权
- **JWT**：商业版高级功能，对接现代登录系统

你接下来想看哪一节？
- Basic HTTP 认证（最简单）
- auth_request 子请求认证（最常用、最实用）
- 还是 JWT 介绍（商业版）
