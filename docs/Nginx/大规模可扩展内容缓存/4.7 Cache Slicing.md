# 4.7 缓存分片（Cache Slicing）· 终极通俗解释
我用**最简单、最直观、不绕弯**的方式给你讲透👇

# 核心一句话
**slice = 把大文件（视频/安装包/ISO）切成一小块一小块缓存（比如 1MB 一块）。
好处：只缓存用户需要的片段，不缓存整个大文件，节省空间、提升效率、支持断点续传。**

---

# 1. 先看配置（逐句翻译）
```nginx
proxy_cache_path /tmp/mycache keys_zone=mycache:10m;

server {
    proxy_cache mycache;
    slice 1m;               # 切成 1MB 一片
    proxy_cache_key $host$uri$is_args$args$slice_range;  # 每片一个独立缓存key
    proxy_set_header Range $slice_range; # 向后端请求对应片段
    proxy_cache_valid 200 206 1h; # 缓存正常和分片响应
    proxy_http_version 1.1; # 必须用1.1，支持分片请求

    location / {
        proxy_pass http://origin:80;
    }
}
```

---

# 2. 什么是“切片”？
不加 slice：
- 用户请求视频 0:00 ~ 0:10
- Nginx **下载整个视频**
- 缓存**整个视频**
- 浪费巨大磁盘 + 带宽

加 slice 1m：
- 视频切成 **1MB 一片**
- 用户看哪片 → Nginx 只下哪片
- 只缓存**那一片**
- 超级省空间、超快

---

# 3. 核心变量：$slice_range
这个是 Nginx **自动生成**的，代表当前片段范围：
```
bytes=0-1048575
bytes=1048576-2097151
...
```

- 每一片有独立的缓存 key
- 每一片独立缓存
- 每一片不会互相干扰
- 支持**断点续传、拖动播放、部分下载**

---

# 4. 为什么必须 HTTP/1.1？
因为 **HTTP/1.0 不支持 Range 分片请求**
所以必须加：
```nginx
proxy_http_version 1.1;
```

---

# 5. 这个功能专门用来干嘛？
- **视频点播（HTML5 Video）**
- **大文件下载**
- **软件包、镜像、ISO**
- **音频、大图片**
- **用户只下载一部分的内容**

---

# 6. 超级重要的注意事项
### **slice 只能用在“不变的大文件”上！**
- 视频
- 安装包
- 静态资源

如果文件会变：
- Nginx 会检查 `ETag`
- 文件变了 → 缓存失效，重新下载

---

# 7. 和 cache_lock 的关系
- 文件**不变** → 用 **slice**
- 文件**会变** → 用 **cache_lock**

---

# 超精简总结（必须记住）
**slice = 大文件切小块缓存
只缓存用户需要的部分
省磁盘、省带宽、支持视频拖动
专门用于：视频、大文件、静态资源**
