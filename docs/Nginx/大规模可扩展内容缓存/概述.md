**Massively Scalable Content Caching（大规模可扩展内容缓存）** 是 NGINX 的核心高性能能力，核心是通过**分层、分布式、高吞吐的缓存架构**，让静态/动态内容直接从边缘或代理层返回，**彻底减轻后端服务器压力**，并支持**百万级QPS**横向扩展。

---

## 一、核心作用（一句话）
- **缓存后端响应**，直接由 NGINX 磁盘/内存返回
- **降低延迟 10~1000 倍**、**后端负载下降 80%+**
- 支持**水平扩展集群**，自建类 CDN 能力

## 二、基础配置（必掌握）
```nginx
http {
    # 定义缓存存储区（http层全局）
    proxy_cache_path /var/cache/nginx/content
        levels=1:2          # 两级目录（防单目录文件过多）
        keys_zone=CONTENT_CACHE:100m  # 内存元数据区（100MB）
        max_size=50g        # 磁盘缓存最大50GB
        inactive=7d         # 7天未访问自动删除
        use_temp_path=off;  # 直接写入缓存（性能更高）

    # 全局默认缓存规则
    proxy_cache_key "$scheme$request_method$host$request_uri";
    proxy_cache_valid 200 301 302 1h;  # 成功响应缓存1小时
    proxy_cache_valid 404      1m;      # 404缓存1分钟
    proxy_cache_valid any      10s;     # 其他缓存10秒

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend;
            proxy_cache CONTENT_CACHE;   # 启用缓存

            # 缓存状态头（调试用）
            add_header X-Cache-Status $upstream_cache_status;
        }
    }
}
```

### 关键参数解释
- **levels=1:2**：缓存文件散列到两级目录（如 `/c/ab/xxx`），避免单目录超量文件
- **keys_zone**：共享内存存缓存元数据（1MB ≈ 8000 条）
- **max_size**：磁盘缓存上限，满后**LRU 自动清理**最久未访问内容
- **inactive**：超时未访问自动淘汰
- **X-Cache-Status**：返回 HIT / MISS / STALE（监控必备）

## 三、大规模扩展的关键机制（可扩展核心）
### 1. 缓存锁：防“惊群效应”（Thundering Herd）
```nginx
proxy_cache_lock on;           # 同一缓存键只放行1个请求到后端
proxy_cache_lock_age 5s;       # 锁超时5秒
proxy_cache_lock_timeout 10s;
```
- 高并发下**避免大量请求同时穿透到后端**

### 2.  stale 机制：后端宕机仍可用
```nginx
proxy_cache_use_stale error timeout updating http_500 http_503;
proxy_cache_background_update on;  # 后台异步刷新缓存
```
- 后端故障/超时/更新中，**继续返回旧缓存**，服务不中断

### 3. 微缓存（Microcaching）：动态内容秒级缓存
```nginx
proxy_cache_valid 200 1s;  # 动态页面缓存1秒
```
- 适合新闻、论坛、商品列表等高动态页面
- **1秒缓存可扛 100倍QPS**，同时数据基本实时

### 4. 分片缓存（Slice Caching）：大文件高效缓存
```nginx
location /video/ {
    proxy_cache VIDEO_CACHE;
    slice 1m;               # 1MB分片
    proxy_cache_key $uri$is_args$args$slice_range;
    proxy_set_header Range $slice_range;
    proxy_pass http://storage;
}
```
- 大文件（视频/安装包）分片缓存，**支持断点续传、并发读**

### 5. 多区域分层缓存
```nginx
# 商品详情（大缓存、长时效）
proxy_cache_path /cache/product ... keys_zone=PRODUCT:200m max_size=100g inactive=24h;
# 首页（小缓存、短时效）
proxy_cache_path /cache/index ... keys_zone=INDEX:50m max_size=10g inactive=5m;

location /product/ { proxy_cache PRODUCT; }
location /index/   { proxy_cache INDEX; }
```
- 按业务场景**分池缓存**，命中率、性能最优

## 四、横向扩展：构建缓存集群（真正大规模）
1. **多层级缓存架构**
   - 边缘层（NGINX 边缘节点）→ 中心层（NGINX 缓存集群）→ 源站
   - 越靠近用户，缓存命中率越高、延迟越低

2. **一致性哈希负载均衡**
```nginx
upstream cache_cluster {
    hash $request_uri consistent;  # 一致性哈希
    server cache1.example.com;
    server cache2.example.com;
}
```
- 相同 URI 固定落到同一缓存节点，**命中率最大化**

3. **缓存清除（Purge）**
```nginx
location ~ /purge(/.*) {
    allow 127.0.0.1;
    deny all;
    proxy_cache_purge CONTENT_CACHE "$scheme$request_method$host$1";
}
```
- 主动清理更新/失效内容
