# 10.1 自动部署（Auto-Provisioning）—— 云端自动化搭建 Nginx 服务器
我给你**用人话、极简、100% 听懂**的方式讲👇

---

# 核心一句话
**自动部署 = 让云服务器（AWS EC2）开机时
自动安装 Nginx → 自动配置 → 自动启动
全程不用人工操作，机器自己搞定。**

这是**云上自动化运维**的基础。

---

# 这一节讲了 3 个关键技术
## 1. **AMI 镜像**（亚马逊机器镜像）
- 相当于**服务器的“备份包/模板”**
- 你可以把装好 Nginx 的系统做成一个镜像
- 以后**一键开出几百台一模一样的服务器**

## 2. **EC2 UserData**
- 服务器**第一次开机时自动运行的脚本**
- 可以写：安装软件、下载配置、启动服务、修改环境
- 像自动化的“开机自启动任务”

## 3. **自动化工具**
- **Ansible / Chef**：配置管理（定义服务器状态）
- **Packer**：自动制作 AMI 镜像（全自动打包）

---

# 三种部署方式（最重要）
## ① 开机再安装（最慢）
开机 → 联网下载 Nginx → 配置  
**缺点：慢、容易失败**

## ② 完整镜像（最快）
把 Nginx + 配置全部打包进 AMI  
开机直接用  
**缺点：不灵活，改配置要重新做镜像**

## ③ 部分镜像（推荐！混合最佳）
- 镜像里**只装 Nginx 软件**
- **配置文件开机时通过 UserData 自动生成**

✅ **速度快**  
✅ **灵活**  
✅ **企业最常用**

---

# 工作流程（超级清晰）
1. 用 **Packer + Ansible** 制作带 Nginx 的 AMI  
2. 启动 EC2  
3. **UserData 自动运行**，写入环境配置  
4. Nginx 自动加载配置、启动服务  
5. **服务器直接上线，无需人工干预**

---

# UserData 能干嘛？
- 下载 Nginx 配置
- 设置域名
- 配置上游服务器
- 注册到负载均衡
- 设置环境变量

一句话：  
**UserData = 开机自动做配置**

UserData = 开机自动配置脚本  
Packer/Ansible = 自动化制作模板**
