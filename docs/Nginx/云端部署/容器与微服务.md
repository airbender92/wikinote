# 容器与微服务（Containers/Microservices）
## 超精简人话总结

这一章讲的是：**把 Nginx 放进容器（Docker），并在 Kubernetes / 微服务架构里使用**。

---

## 1. 容器带来的核心变化
- 把**依赖、环境、配置、软件**全部打包在一起
- 开发、测试、生产环境**完全一致**
- 避免“我本地能跑，服务器不行”
- 部署极快、可移植、可弹性扩缩

## 2. 为什么要把 Nginx 容器化？
- 微服务架构中，网关、负载均衡、代理**也要容器化**
- Nginx 镜像小、启动快、极适合容器
- 统一整个技术栈：全部跑在容器里

## 3. 本章主要内容
- 构建 Nginx 容器镜像
- 容器环境下的 Nginx 优化配置
- 在 **Kubernetes、OpenShift** 上部署 Nginx
- 用 Nginx 做 **API Gateway**（路由、安全、认证、限流）

## 4. 容器化 Nginx 的两个重要注意点
### ① 日志必须输出到标准输出
```
/dev/stdout   正常日志
/dev/stderr   错误日志
```
这样 Docker/K8s 日志驱动才能收集。

### ② 负载均衡策略（Nginx Plus）
- `least_time` 最适合容器网络
- 优先选择**响应最快**的上游服务
- 自动就近选择网络延迟最低的实例

## 5. Nginx as API Gateway
在微服务里，Nginx 充当统一入口：
- 请求路由
- 认证鉴权
- HTTPS 加密
- 限流、缓存
- 安全过滤
