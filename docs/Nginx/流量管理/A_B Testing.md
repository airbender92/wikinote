# 3.1 A/B 测试 · 超级通俗解释
我给你**用最简单、最生活化、不绕弯**的方式，一次性讲透👇

# 核心一句话
**A/B 测试 = 把一部分用户分给新版本，另一部分分给旧版本，对比哪个效果更好。**
Nginx 用 `split_clients` 模块**按百分比分流**，实现无缝测试。

---

# 1. 最核心配置（一看就懂）
```nginx
split_clients "${remote_addr}AAA" $variant {
    20.0%  "backendv2";  # 20% 用户 → 新版本
    *      "backendv1";  # 剩下 80% → 旧版本
}
```

### 逐行解释
- **`split_clients`**：Nginx 专门用来**按比例分流**的指令
- **`${remote_addr}AAA`**：用用户 IP + 字符串做哈希，保证**同一个用户永远分到同一版本**
- **`$variant`**：生成一个变量，值是 `backendv1` 或 `backendv2`
- **`20.0%`**：20% 用户用新版本
- **`*`**：剩下所有用户（80%）用旧版本

---

# 2. 怎么用这个分流？
```nginx
location / {
    proxy_pass http://$variant;  # 自动分给不同服务器组
}
```
- 20% 请求 → `backendv2`（新版本）
- 80% 请求 → `backendv1`（旧版本）

---

# 3. 还能分流静态网站
```nginx
split_clients "${remote_addr}" $site_root_folder {
    33.3%  "/var/www/sitev2/";
    *      "/var/www/sitev1/";
}

server {
    root $site_root_folder;  # 不同用户看到不同文件夹的页面
}
```

---

# 4. 业务用途（超级重要）
- **A/B 测试**：对比两个页面/功能哪个转化率高
- **金丝雀发布（Canary Release）**：先放 5% 流量给新版本，没问题再全量
- **蓝绿部署**：新版本验证没问题，一键切 100% 流量
- **降低风险**：出问题只影响一小部分用户

---

# 5. 关键特点
✅ **同一个用户永远分到同一个版本**（不会来回跳）
✅ **按任意百分比分流**（1%、5%、10%、50% 都行）
✅ 可以分流**后端服务**或**静态页面**
✅ 开源版 Nginx 直接能用

---

# 超精简总结（必须记住）
**split_clients = 按百分比分流用户
同一个用户永远不变
用于 A/B 测试、金丝雀发布、蓝绿发布**
