# Traffic Management —— 流量管理
大白话解释：
**就是用 Nginx 对进来的请求做精细控制：分流、限流、跳转、改写、黑白名单、限速、路由规则等。**

---

## 简单说，它管这几件事：
1. **流量分发**
   不同路径 / 域名 / 设备，分到不同后端服务
2. **流量控制**
   限制访问频率、并发连接，防爬虫、防攻击
3. **流量改写**
   修改 URL、请求头、重定向、HTTPS 强制跳转
4. **流量过滤**
   允许/拒绝某些 IP、地区、UA 访问
5. **灰度/蓝绿发布**
   只给一部分人发新版本，不影响全部用户

---

## 和前面“负载均衡”的区别
- **负载均衡**：把流量分给多台服务器，保证不压垮
- **流量管理**：更上层，控制**谁能进、进哪、怎么进、进多少**

一句话总结：
**负载均衡是“怎么分”，流量管理是“怎么管”。**
