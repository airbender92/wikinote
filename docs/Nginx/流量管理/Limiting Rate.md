# 3.6 速率限制（Limiting Rate）**终极通俗解释**
我直接用**最清晰、最不绕弯**的方式讲透，你马上就能懂👇

# 核心一句话
**limit_req = 限制“每秒能发多少个请求”**
这才是你刚才问的 **“限制请求个数”**！

---

# 1. 先看基础配置（最常用）
```nginx
http {
    # 定义限流规则：按IP，10M内存，每秒最多3个请求
    limit_req_zone $binary_remote_addr zone=limitbyaddr:10m rate=3r/s;

    # 超限返回 429（请求过多）
    limit_req_status 429;

    server {
        # 开启限流
        limit_req zone=limitbyaddr;
    }
}
```

### 逐句解释
- `rate=3r/s` = **每秒只允许 3 个请求**
- 超过这个速度 → **直接返回 429 拒绝**
- 按 **IP** 限制

这就是 **真正限制请求次数** 的配置！

---

# 2. 你最关心的问题：
## **limit_conn（连接） vs limit_req（速率） 到底区别是什么？**

### ✅ **limit_conn 限制连接数**
- 限制**同时有多少个连接在线**
- 适合：下载、长连接、WebSocket

### ✅ **limit_req 限制请求速率**
- 限制**每秒发多少个请求**
- 适合：API、网页、登录接口、防攻击
- **这才是限制“请求个数”**

---

# 3. 进阶：burst（突发流量）+ delay（延迟）
```nginx
limit_req zone=limitbyaddr burst=12 delay=9;
```

### 大白话解释：
- `rate=3r/s`：正常每秒 3 个请求
- `burst=12`：**最多允许突发 12 个请求排队**
- `delay=9`：**前 9 个不卡，后面 3 个排队延迟**
- 超过 12 个 → **直接拒绝 429**

### 简单理解：
- 正常速度：**3 个/秒**
- 突然来了一波请求：
  - 前 9 个：**立刻处理**
  - 接下来 3 个：**排队慢慢处理**
  - 超过 12 个：**拒绝**

---

# 4. nodelay 模式（更粗暴）
```nginx
limit_req zone=limitbyaddr burst=12 nodelay;
```
- 突发 12 个请求**一次性全部处理**
- 处理完后必须等时间恢复额度
- 适合允许瞬间爆发，但不能一直爆

---

# 5. 这个功能用来干嘛？
- **防暴力破解登录**（登录页严格限流）
- **防爬虫**
- **防CC攻击**
- **保护API不被刷爆**
- **公平分配资源**

---

# 6. 还有一个安全测试模式：dry_run
```nginx
limit_req_dry_run on;
```
- **不会真拦截**
- 只在日志里记录：PASSED / REJECTED
- 你可以线上观察流量，**调好了再真正开启拦截**
- 超级安全，不会误伤用户

---

# 超级精简总结（最重要）
### **limit_req = 限制请求速率（次/秒）**
- `rate=3r/s` → 每秒 3 个请求
- `burst=12` → 突发排队
- `delay=9` → 前9个不卡
- `429` → 请求过多
- `dry_run` → 测试不拦截

---
