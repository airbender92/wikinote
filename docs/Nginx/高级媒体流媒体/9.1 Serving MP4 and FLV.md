# 9.1 提供 MP4 和 FLV 视频流媒体服务
## **超通俗、极简解释**

我给你用**最直白、最实用**的方式讲👇

---

# 核心一句话
**给 Nginx 加 2 个指令：
`mp4` 和 `flv`
就能让视频支持**进度条随意拖动**、**边下边播**，不用等整个文件下载完。**

这叫 **伪流媒体（Pseudostreaming）**。

---

# 配置代码（你给的）
```nginx
http {
    server {
        # MP4 视频目录
        location /videos/ {
            mp4;  # 开启 MP4 伪流媒体
        }

        # FLV 视频
        location ~ \.flv$ {
            flv;  # 开启 FLV 伪流媒体
        }
    }
}
```

---

# 这两个指令到底做什么？

## 1. `mp4;`
作用：
- 支持 MP4 视频**边下边播**
- 支持**拖动进度条**跳到未下载的位置
- 支持 `start=` 参数播放指定时间点

## 2. `flv;`
作用：
- 支持 FLV 视频**流式播放**
- 支持进度条拖动

---

# 什么是伪流媒体（Pseudostreaming）？
不用复杂的直播服务器
**只用 HTTP 下载，就能实现：**
✅ 不用等文件下完就能播
✅ 随意拖动进度条
✅ 手机/浏览器全支持

这就是 Nginx 最强大的视频能力！

---

# 为什么 Nginx 适合做视频服务？
- **超高性能**
- **抗高并发**
- **极低资源占用**
- 支持**大视频流畅拖动**
- 不用安装复杂软件，一行指令开启

---

# 最简单总结
```
mp4;  → 让 MP4 支持拖动、边下边播
flv;  → 让 FLV 支持拖动、边下边播
```

**一行指令，视频秒变流媒体！**
