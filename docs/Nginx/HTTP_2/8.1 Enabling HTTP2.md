# 8.1 启用 HTTP/2（最简单、最清晰的解释）
我给你**用人话、不绕弯、100% 听懂**👇

---

# 核心一句话
**HTTP/2 只需要在 listen 443 后面加 `http2 on;` 就开启了！
但必须配合 HTTPS（SSL/TLS）才能用。**

---

# 开启 HTTP/2 的完整配置（你给的）
```nginx
server {
    listen 443 ssl default_server;
    http2 on;  # 开启 HTTP/2

    ssl_certificate     server.crt;
    ssl_certificate_key server.key;
}
```

就这么简单！

---

# 必须知道的 4 个关键点（非常重要）

## 1. HTTP/2 **必须用 HTTPS**
虽然协议本身不强制加密，
但**所有浏览器都只支持 HTTPS + HTTP/2**。

→ **没有 SSL 证书，HTTP/2 等于没开。**

---

## 2. 旧的、弱的加密套件会被 HTTP/2 拒绝
比如：
- TLS 1.0 / 1.1
- 弱加密算法（MD5、SHA1）

但你**不用担心**：
**Nginx 默认配置就是安全的，直接用就行。**

---

## 3. ALPN（应用层协议协商）
HTTPS 握手时，浏览器和 Nginx 自动协商：
- 用 HTTP/2？
- 还是用 HTTP/1.1？

**Nginx 自动处理，不用配置。**

---

## 4. 怎么验证是否开启成功？
### 方法 A：浏览器里看
Chrome → F12 → Network → 协议列
显示 `h2` = 开启成功

### 方法 B：命令行工具
```bash
nghttp -v https://你的域名
```

看到 `HTTP/2` 就说明成功。
