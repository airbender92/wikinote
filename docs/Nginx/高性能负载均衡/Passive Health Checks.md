# 2.9 被动健康检查 · 超通俗解释（一看就懂）
我用**最简单、最直白**的话，把这段内容一次性讲透👇

# 核心一句话
**被动健康检查 = Nginx 不主动探测服务器，而是在用户请求失败时，自动标记服务器为不健康，暂时不转发流量。**
（开源版 Nginx 自带，免费可用）

---

# 配置解释（你给的代码）
```nginx
upstream backend {
  server backend1.example.com:1234 max_fails=3 fail_timeout=3s;
  server backend2.example.com:1234 max_fails=3 fail_timeout=3s;
}
```

### 两个关键参数：
1. **`max_fails=3`**
   连续失败 **3 次**，就认为这台服务器挂了。

2. **`fail_timeout=3s`**
   标记为不健康后，**暂停 3 秒**不发流量给它。
   3 秒后再自动尝试恢复。

---

# 什么是“被动”？
- **不主动发探测包**
- 只在**用户真实请求失败**时才检测
- 失败 → 计数 → 达到次数 → 临时摘除服务器
- 超时后自动重试

这就是**被动健康检查**。

---

# 默认值（不写也会生效）
不配置的话，Nginx 默认使用：
- `max_fails=1`（失败 1 次就摘除）
- `fail_timeout=10s`（暂停 10 秒）

---

# 支持哪些负载均衡？
全部支持 ✅
- HTTP
- TCP
- UDP

---

# 超精简总结
**被动健康检查 = 请求失败算次数 → 达到上限暂时隔离 → 超时自动恢复
开源版 Nginx 自带，不用额外配置，稳定可靠。**
