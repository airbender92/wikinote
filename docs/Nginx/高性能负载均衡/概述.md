# 高性能负载均衡（High-Performance Load Balancing）
用最简单、最直白的话解释：

**让大量用户请求，均匀、高效、不卡顿地分摊到多台后端服务器上，同时保证速度快、并发高、稳定不掉线。**

---

## 拆开理解
- **Load Balancing（负载均衡）**
  流量来了不压在一台机器上，分给多台服务器一起处理，避免单点崩溃、访问变慢。

- **High-Performance（高性能）**
  Nginx 本身处理请求极快、占用资源少，能同时扛住极高并发，延迟极低，这就是“高性能”。

---

## 合起来一句话
**高性能负载均衡 = 用 Nginx 快速、高效地把流量分给多台服务器，实现速度快、压力匀、服务稳。**

这就是 Nginx 最核心、最出名的用途之一。


# 大白话整体解释
做负载均衡时，必须保证**用户体验只变好、不受影响**。

现在很多应用是**无状态**的，用户数据存在共享缓存或数据库里，请求分到哪台服务器都一样。
但还有很多应用是**有状态**的，用户会话数据只存在某一台后端服务器上，比如大型数据处理、会话信息存在本地的场景。

这种情况下，**同一个用户的后续请求必须还分到同一台服务器**，不然会话就失效、用户要重新登录、数据丢失。
而且这台服务器要等用户会话结束才能下线。

Nginx 提供了**会话保持**功能（通过 Cookie 或路由规则），保证同一个用户始终访问同一台后端机器。
本章就是讲 Nginx 如何实现负载均衡下的会话保持。

---

# 超精简总结
- 无状态应用：随便分服务器都行
- 有状态应用：必须让同一个用户一直访问同一台服务器
- 这就叫**会话保持**，Nginx 用 Cookie 等方式实现，本章讲这个。
