# 2.4 负载均衡算法 · 超级通俗解释
我给你**用最简单、最生活化、一眼全懂**的方式，把 Nginx 所有负载均衡策略一次性讲透！
不用记专业名词，看完就会。

---

# 核心一句话
Nginx 提供 **6 种负载均衡算法**，你可以根据业务场景选最合适的分发方式。
默认是**轮询（round-robin）**，但其他算法更智能。

---

# 6 种算法大白话解释（超易懂）

## 1. **轮询 Round Robin（默认）**
- 挨个发请求：1 → 2 → 1 → 2
- 可以加 `weight` 权重
- 最简单，但不看服务器压力

## 2. **最少连接 Least Connections（least_conn）**
- **谁空闲分给谁**
- 哪台服务器当前连接最少，请求就发给它
- 适合请求处理时间长短不一的场景

## 3. **最少时间 Least Time（least_time，仅 Nginx Plus 商业版）**
- **选连接最少 + 响应最快**的服务器
- 最智能、性能最好
- 看响应时间，不只看连接数

## 4. **通用哈希 Hash（hash key）**
- 根据你指定的特征（IP、URL、Cookie 等）做哈希
- 相同特征 → 永远分给同一台服务器
- **HTTP/TCP/UDP 都能用**

例子：
```
hash $remote_addr;
```
同一个IP → 固定一台服务器

## 5. **随机 Random**
- 随机选一台
- 可配置随机选两台，再从中挑最优（默认最少连接）

## 6. **IP 哈希 IP Hash（ip_hash，仅 HTTP）**
- 按客户端 IP 分配固定服务器
- 保证**会话保持**，用户永远访问同一台后端
- **只能用于 HTTP**，不能用于 TCP/UDP

---

# 最重要区别（必须记住）
- **ip_hash**：仅 HTTP，按IP固定会话
- **hash $remote_addr**：HTTP/TCP/UDP 都能用，效果一样
- **least_conn**：谁闲分给谁
- **least_time**：商业版，选最快的
- **round robin**：默认挨个发
- **random**：随机发

---

# 最实用场景总结
- 网站会话保持 → **ip_hash 或 hash**
- 接口耗时不一 → **least_conn**
- 追求最高性能 → **least_time（商业版）**
- 简单分发 → **round robin**

---

# 超精简总结
**Nginx 负载均衡 = 6 种算法
轮询、最少连接、最少时间、哈希、随机、IP哈希
按需选择，让流量分发更合理、服务更稳、速度更快。**
