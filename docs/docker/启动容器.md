你这条命令：

```bash
docker exec -it my-nginx /bin/bash
```

报错：

> `container ... is not running`

说明：**容器 `my-nginx` 已经停止了，所以你进不去。**

---

## 一、先确认容器状态

在终端执行：

```bash
docker ps -a
```

你会看到类似：

```
CONTAINER ID   IMAGE     COMMAND                  STATUS                      NAMES
96cd791bdd...  nginx     "/docker-entrypoint.…"   Exited (0) 2 minutes ago   my-nginx
```

- 如果 `STATUS` 是 `Exited`，就表示容器没在运行
- 所以 `docker exec` 会报 `is not running`

---

## 二、让容器先跑起来

如果容器是 `Exited` 状态，先启动它：

```bash
docker start my-nginx
```

再检查：

```bash
docker ps
```

如果 `STATUS` 变成 `Up x minutes`，说明容器已经运行。

---

## 三、再执行 exec 进入容器

```bash
docker exec -it my-nginx /bin/bash
```

如果镜像里没有 `/bin/bash`，可以试试：

```bash
docker exec -it my-nginx /bin/sh
```

---

## 四、如果容器总是自己退出

你之前用：

```bash
docker run -d -p 80:80 --name my-nginx nginx
```

正常情况下，这个命令会**持续运行**。

如果它很快 `Exited`，可能原因有：

- 端口被占用：  
  ```bash
  netstat -ano | findstr :80
  ```
- 资源不足
- 镜像拉取不完整

可以重新拉取并运行：

```bash
docker stop my-nginx
docker rm my-nginx
docker pull nginx
docker run -d -p 80:80 --name my-nginx nginx
```

---

## 五、小结

- `docker exec` 只能对 **正在运行的容器** 使用
- 先 `docker start` 或 `docker run` 让容器运行
- 再 `exec` 进入
- 如果容器总退出，要查日志：  
  ```bash
  docker logs my-nginx
  ```
