# 核心问题
Docker 里的 nginx 镜像 **没有 vi 编辑器**，所以报错。

# 解决方法（直接复制运行）
用 **`cat` 命令直接覆盖写入文件**，全程不用编辑器！

## 直接运行这一整段命令（复制粘贴）
```bash
cat > default.conf << EOF
server {
 listen 80 default_server;
 server_name www.example.com;
 location / {
 root /usr/share/nginx/html;
 index index.html index.htm;
 }
}
EOF
```

## 然后检查是否写入成功
```bash
cat default.conf
```

## 最后生效配置
```bash
nginx -t && nginx -s reload
```

# 这段命令到底做了什么？
1. 直接**创建/覆盖** `default.conf`
2. 把书里要求的配置**完整写进去**
3. 不用 vi、不用 nano、不用装任何工具

# 逐段大白话解释

### 1. `cat > default.conf << EOF`
- `cat`：查看/写入文件的工具
- `> default.conf`：**把内容写入 default.conf 这个文件**
- `<< EOF`：开始标记，意思是**接下来直到 EOF 为止，都是要写入的内容**

### 2. 中间这一段
```nginx
server {
 listen 80 default_server;
 server_name www.example.com;
 location / {
 root /usr/share/nginx/html;
 index index.html index.htm;
 }
}
```
这就是**书里要求你写的 Nginx 配置**，会原封不动写进文件里。

### 3. 最后一个 `EOF`
表示**内容结束**，命令停止写入，保存文件。
