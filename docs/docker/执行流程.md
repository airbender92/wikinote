## 一、`/docker/setenv.sh` 是什么？什么时候执行？

### ✅ 结论先给

> **`setenv.sh` 不是 Nginx 自动执行的**  
> **也不是 npm 自动执行的**  
> ✅ **它是你在 Dockerfile / docker run / entrypoint 里“人为安排”执行的**

---

## 二、常见执行时机（按真实项目）

### ✅ 场景 1：Dockerfile 里执行（最常见）

```dockerfile
COPY docker/setenv.sh /docker/setenv.sh
RUN chmod +x /docker/setenv.sh
RUN /docker/setenv.sh
```

📌 执行时机：
- **镜像构建阶段（docker build）**
- 只执行一次

✅ 适合：
- 生成 nginx 配置
- 替换环境变量
- 初始化配置文件

---

### ✅ 场景 2：容器启动时执行（ENTRYPOINT / CMD）

```dockerfile
ENTRYPOINT ["/docker/setenv.sh"]
CMD ["nginx", "-g", "daemon off;"]
```

📌 执行时机：
- **每次 docker run**
- 容器启动前 / 启动时

✅ 适合：
- 根据环境变量动态生成 `server {}`
- 区分 dev / test / prod

---

### ✅ 场景 3：手动执行（调试）

```bash
docker exec -it my-nginx sh
/docker/setenv.sh
```

📌 执行时机：
- 你想改配置但不重建镜像

---

## 三、`npm run build` 和 `setenv.sh` 的关系

### ✅ 前端构建流程（标准）

```text
npm install
npm run build
↓
dist / build 目录
↓
复制到 nginx 容器
```

### ✅ 典型 Dockerfile（前后端分离）

```dockerfile
# 构建阶段
FROM node:18 AS build
WORKDIR /app
COPY . .
RUN npm install && npm run build

# 运行阶段
FROM nginx
COPY --from=build /app/dist /usr/share/nginx/html
COPY docker/setenv.sh /docker/setenv.sh
RUN /docker/setenv.sh
```

📌 执行顺序：

```
1️⃣ npm run build
2️⃣ 生成静态文件
3️⃣ setenv.sh 生成 nginx 配置
4️⃣ nginx 启动
```

---

## 四、`setenv.sh` 里通常写什么？

你之前看到的：

```bash
cat >> /etc/nginx/conf.d/default.conf <<EOF
server {
    listen 80;
    server_name localhost;
}
EOF
```

✅ 常见用途：
- 动态写入 `server {}`
- 替换 `${API_BASE_URL}`
- 根据环境变量切换配置

---

## 五、前端 `npm run build` 本身会触发什么？

### ❌ 不会触发：
- Docker
- nginx
- setenv.sh

### ✅ 只做一件事：
> **把源码打包成静态文件**

例如：

```text
dist/
├── index.html
├── assets/
```

## 六、完整真实世界流程（强烈建议记住 ✅）

```text
开发
  ↓
npm run build
  ↓
Dockerfile COPY dist → nginx
  ↓
docker build
  ↓
setenv.sh 生成 nginx 配置
  ↓
docker run
  ↓
nginx 提供服务
```

---

## 七、一句话总结（重点 ✅）

> ✅ `setenv.sh` **只在 Docker 构建或容器启动时执行**  
> ✅ `npm run build` **只负责前端打包，不碰 Docker / nginx**  
> ✅ 两者通过 **Dockerfile 串联**
