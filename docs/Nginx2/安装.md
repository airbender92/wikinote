# 译文：NGINX 开源版两个版本 + 从系统仓库安装步骤

## NGINX 开源版有两个版本
**Mainline（主线版）**
- 最新、功能最全，包含最新特性和 Bug 修复
- 适用场景：想要最新功能、改进的用户
- 说明：整体稳定，但可能包含实验性功能

**Stable（稳定版）**
- 不一定有最新功能，但包含关键 Bug 修复
- 适用场景：**生产环境强烈推荐**，追求稳定、可靠

---

## 安装方式（二选一）
1. 从 Ubuntu / Debian 默认系统仓库安装
2. 从 nginx.org 官方仓库安装

---

## 从系统仓库安装预编译包
### 第 1 步：更新软件包列表
安装任何新软件前，最好先更新包列表，确保安装最新版本。

在终端执行：
```bash
sudo apt-get update
```
### 第 2 步：安装 NGINX
```bash
sudo apt-get install nginx
```
### 第 3 步：检查安装版本
```bash 
sudo nginx -v
```

# 译文：在 Docker 中运行 Nginx
Docker 是一个容器化平台，让开发者和系统管理员可以把应用打包、分发，并运行在轻量、可移植的容器里。如今科技公司大量使用 Docker，而在 Docker 容器中运行开源版 Nginx 也是非常普遍的做法。

在容器中运行 Nginx 的步骤如下：

## 前提条件
你的 Windows 或 Linux 系统已安装 Docker。

---

## 第 1 步：拉取 Nginx 镜像
```bash
docker pull nginx
```

## 第 2 步：启动 Nginx 容器
```bash
docker run -d -p 80:80 --name mynginx nginx
```

这条命令的作用：
- **-d**：后台运行容器
- **-p 80:80**：将容器的 80 端口映射到宿主机 80 端口
- **--name mynginx**：给容器起一个名字叫 mynginx
- **nginx**：使用的镜像名称

## 第 3 步：访问 Nginx
打开浏览器访问：
- `http://localhost`
- 或 `http://你的服务器IP`

你会看到 Nginx 默认欢迎页。

---

## 常用 Docker 管理命令
1. 停止容器
```bash
docker stop mynginx
```
2. 启动容器
```bash
docker start mynginx
```
3. 删除容器
```bash
docker rm mynginx
```
4. 删除 Nginx 镜像
```bash
docker rmi nginx
```

---

## 重要说明
以上只是基础步骤。实际使用中你可能需要重新配置，例如：
- 使用 SSL 证书
- 配置反向代理
- 设置高级 Nginx 功能

建议参考官方 Nginx Docker Hub 页面获取最新信息和最佳实践。

---

## 重要提示
无论用哪种方式安装 Nginx，默认配置文件都在：
`/etc/nginx/`

主要文件包括：
- `conf.d/`
- `nginx.conf`
- `mime.types`

这个目录是你后续所有 Nginx 配置的**主要工作区**。

# 翻译正文
作为 Nginx 管理员，你会经常更新和创建 Nginx 配置。以下命令将是你最主要的工具。Nginx 在服务器中作为服务运行，始终在后台保持活跃。

1. **启动 Nginx 服务**  
   用于启动 Nginx。

2. **停止 Nginx 服务**  
   用于停止 Nginx。

3. **重启 Nginx 服务**  
   重启命令会关闭正在运行的 Nginx 并重新启动。  
   仅在执行重大配置变更（如修改端口、网络接口）时使用。  
   该命令会强制关闭所有工作进程。

4. **重载 Nginx 服务**  
   这是你日常工作中**最重要**的命令。  
   重载命令会在不中断服务的情况下加载新配置，不会关闭旧连接。  
   如果新配置文件存在语法错误，重载会被中止，服务器继续使用旧配置。  
   ✅ **注意：重载比重启更安全。每次修改配置后都必须执行重载才能生效。**

5. **查看 Nginx 服务状态**  
   检查 Nginx 是否正在运行。  
   - 运行中：显示绿色 active (running)  
   - 未运行：显示 inactive  
   - 启动失败：显示红色 failed

---

### 额外两个常用命令：`nginx -t` 和 `nginx -T`
**nginx -t**  
用于**测试配置文件语法**，不会重启或重载服务。  
在执行 reload 之前，强烈建议先运行该命令检查语法是否正确。

**nginx -T**  
Nginx 配置可以分散在多个 `.conf` 文件中，通过 `include` 引入主配置。  
`nginx -T` 会**合并所有配置文件**，并在终端输出完整的最终配置，方便调试、校验与排错。

# 🚀 **最重要：Docker 环境下的等效命令（直接用）**
假设你的容器名称是 `mynginx`

### 1. 测试配置语法（最常用！）
```bash
docker exec mynginx nginx -t
```

### 2. 平滑重载配置（不会断连接）
```bash
docker exec mynginx nginx -s reload
```

### 3. 重启 Nginx（会中断连接）
```bash
docker restart mynginx
```

### 4. 停止 Nginx
```bash
docker stop mynginx
```

### 5. 启动 Nginx
```bash
docker start mynginx
```

### 6. 查看完整合并后的 Nginx 配置（调试神器）
```bash
docker exec mynginx nginx -T
```

### 7. 查看容器运行状态
```bash
docker ps | grep mynginx
```
